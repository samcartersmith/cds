This major update embraces modern standards, optimizes performance, and improves developer experience. Since the first PR<sup>[1](https://github.cbhq.net/mono/repo/pull/5292)</sup> more than 4 years ago, over 85 different developers have contributed to CDS. Today CDS is home to more than 400 React components across native and web, and CDS code is imported more than 75,000 times<sup>[2](https://sourcegraph.cbhq.net/search?q=context:global+count:all+-repo:%5Egithub%5C.cbhq%5C.net/frontend/cds%24+%22from+%27%40cbhq/cds-%22+OR+%27from+%22%40cbhq/cds-%27+OR+%22require%28%27%40cbhq/cds-%22+OR+%27require%28%22%40cbhq/cds-%27&patternType=keyword&sm=0)</sup> across 96 different repositories.

With v7 we continue to support the evolving needs of our product teams - making developing with CDS faster and safer while preparing for an open-source future.

If your app supports ECMAScript Modules, upgrading CDS should be as simple as installing the new versions!

### Migrating to v7

CDS v7 should work out of the box if your app supports ECMAScript Modules and you're using Coinbase paved roads like Vite, Webpack, Expo, or NextJS. Simply make sure you're using the latest versions of Coinbase dependencies like `@cbhq/vite-config`, `@cbhq/webpack-config`, `@cbhq/jest-config`, etc.

### Troubleshooting

If you experience any trouble migrating we're ready to help! Just reach out in [#ask-ui-systems](https://coinbase.enterprise.slack.com/archives/C05H922EYP7)

### ECMAScript Modules

CDS is now exclusively distributed as ECMAScript Modules.

ECMAScript Modules is the official standard format for JavaScript code reuse, enabling tree shaking and a much faster development environment. This change aligns us with the future of the JavaScript ecosystem and improves compatibility with modern build tools. We do not provide support for CommonJS due to the inherent risks associated with transpiling ECMAScript Modules to CommonJS.

### React 18.3

CDS now supports React 18.3 to ensure compatibility with the upcoming React 19 release. This update reinforces our commitment to keeping CDS ready for the future, allowing developers to leverage the latest React features.

### React Native 0.74 and Expo 51

Upgrading to React Native 0.74 enables bridgeless mode, which is required for the React Native New Architecture. Expo 51 replaces Flipper with Expo dev tools, and adds support for a new Metro resolver which makes bundling up to 6x faster.

### TypeScript 5.6

We've upgraded from TypeScript 5.2 to 5.6, bringing large improvements in type inference and speed. Type checking on CDS source code is now significantly more accurate - allowing us to locate and fix several complex type issues with polymorphic components.

### Looking forward

This release represents a huge step forward for CDS, and we’re committed to continuously improving. These updates lay the groundwork for an exciting, open-source future. The upcoming CDS v8 will include more significant API changes to enable advanced theming capabilities and massive performance improvements. See the [CDS Infra Innovation doc](https://docs.google.com/document/d/1u4UdAiylurXUoUV6xOjoqlApypFIJGFj8mA22j9MoiE) for more info on changes coming in v8.
