## Using the CDS npm packages

1. Add `@cbhq:registry=https://registry-npm.cbhq.net` to your `.npmrc`.
2. Install the CDS packages you need. You can lookup the latest packages here: https://cds.cbhq.net/category/changelog.
   Unfortunately you currently have to use the package version otherwise you may end up installing an alpha version of the library that is tagged as latest.

These are examples, please replace the versions with the latest cds packages when installing.

```bash
yarn add @cbhq/cds-fonts@^0.5.3

# if using web components and your environment has a css-loader for node_modules
yarn add @cbhq/cds-web@^0.40.4

# if using web components and your environment does not have a css loader
# for node_modules (nextjs, Remix etc.)
yarn add @cbhq/cds-web@^0.40.4-esm.0

yarn add @cbhq/cds-mobile@^0.44.1 # if using mobile components
```

## Building with CDS

1. Use either a or b to import the CDS styles into the entry point of your web application

   a. If your environment has a css-loader for node_modules:

   Import `globalStyles` at the root of your application. This includes font face declarations and focus-visible polyfill.
   This should be included at the very top of your imports before any CDS components are imported.

   ```tsx
   import '@cbhq/cds-web/globalStyles';
   ```

   b. If your environment does not have a css loader for node_modules (nextjs, Remix etc.):

   Import `assets/main.css` in the entry point of your application. This will load all styles that are necessary for CDS.
   Import `globalImports.js` in the entry point of your application. This will load the focus-visible polyfill.

2. Add `ThemeProvider` to the root of your React application. You may also add `DevicePreferencesProvider`.
   The latter is optional and only needed if you wish to sync the spectrum and scale based on device settings.

   **Note**: the `ThemeProvider` component will add a `div` to the root of your app. If this is not desired, you can either:

   a. Pass a `display` prop to `ThemeProvider` and set it to `display="contents"`, which will visually remove the ThemeProvider's box and replace it with its content.

   b. Use the `useThemeProviderStyles` hook from `@cbhq/cds-web/system/useThemeProviderStyles` instead and attach the returned `className` and `style` prop to the element of your choice. If you opt to use the `useThemeProviderStyles` hook, you will need to wrap the root of your application with the `FramerMotionProvider` from `@cbhq/cds-web/animation/FramerMotionProvider` the enable animations.

```tsx
import { ThemeProvider } from '@cbhq/cds-web/system/ThemeProvider';
import { DevicePreferencesProvider } from '@cbhq/cds-web/system/DevicePreferencesProvider';

export const App = () => {
  <DevicePreferencesProvider>
    <ThemeProvider>
      <ErrorBoundary>
        <AppRoutes />
      </ErrorBoundary>
    </ThemeProvider>
  </DevicePreferencesProvider>;
};
```

3. Ensure that your html has `<meta charset="UTF-8">` which is required for the icon font in CDS web.

> If you have components that are created outside of the app's main DOM tree, such as `Modal`s that use `createPortal`, you will need to add `ThemeProvider` to the root of the portal element as well.

4. Import components from `@cbhq/cds-web` or `@cbhq/cds-mobile`.
