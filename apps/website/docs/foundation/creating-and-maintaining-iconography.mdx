## Overview

This guideline and contribution model is aimed for non-illustrators to create and contribute back to the system. The guideline will take you through the process of all the things it requires to make an icon. From icon anatomy, tips and tricks, down to stripping back the visual details.

**But first and foremost, we recommend reviewing our Icon Library. We do have an extensive list of icons that you can search on the CDS site or Figma library.**

Using these principles, we recommend understanding the most common approach to the symbol and avoiding re-inventing symbols that entities outside of Coinbase have already defined.

Therefore, we recommend first looking for the most universally understood symbol used when creating a new icon. Second, we suggest looking for a physical, real-world object that you can use to represent the new icon. And finally, we aim to simplify as much as possible.

## Using our template

There is a template provided [here](https://www.figma.com/file/2HNpR1bp11ULksmWKMF9eK/Iconography-Template?node-id=2%3A4); this will be the backbone of your work.

When making icons, you need to create them for all sizes to contribute back to CDS. Once you’re done with the icon designs, please share the Figma link via a request ticket: go/iconrequest, and we will review the work with you!

### Pixel fitting

With each size of the icon, it's essential to align to the pixel grid (the action is also known as ‘pixel fitting’). Whenever possible, snap the shape to x and y coordinates at whole numbers without any decimals.

You can turn on the Pixel Grid tool on Figma (⌘’).

### Diagonals

We use diagonals to aid with our use of arrows and object-related icons.

The choice to use diagonals comes down to two things:

- Meaning: Does the intention change for this icon when shifted on a diagonal? An excellent example of this is the ‘Fullscreen’ icon, as it mimics the motion of the experience going edge to edge on the device.
- Familiarity: Does shifting this visual help with adding the universal appeal of this icon? An excellent example of this is the ‘Tag’ icon; it typically appears on a diagonal rather than straight up.

We also gravitate towards the use of the diagonal to be on the left.

## Style

Our core product iconography style is sharp and bold, always aiming for clarity and trust with solid shapes and using linework sparingly.

### Filled / Solid

We default our icons to be filled with light use of linework unless you are creating in-active and active icons.

### Scaling

We specifically craft every icon for each size in our system.

As we scale down each icon, the details are stripped back, and as we scale up, more details appear. It's all about finding the perfect balance as we create icons. Please do not auto-scale the icons; create each size with care and precision.

If we don’t consider these efforts, we begin to lose visual comprehension of these icons. Scaling is essential to the iconography creation process.

## Alignment

It is not always enough to align an icon to the center of the canvas or even the keyline shape. When there are key geometric shapes, or the icon is unevenly weighted, the icon should make some adjustments to feel like it’s optically centered on the artboard.

### Optical weight

When there is a key geometric shape, the icon should be aligned to the shape's centre and disregard any additional details when finding the optical center.

## Grid is a guide

It's important to note that the grid is a guide; it's not a must follow. Don't use the grid if it's not working for you and the concept you are trying to create. But we do encourage using the keyline shapes!

### Always check the optical balance

Compare and contrast with the other icons in our library. There should be an optical balance with the icons. As much as you may follow the entire guide perfectly, there could be some nuances that don’t work with the rest of the icons.
