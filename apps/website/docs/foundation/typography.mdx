Our typography system consists of Coinbase Sans type family. The sizes of type within the set have been selected for hierarchy and legibility for a range of scales.

We based our definition of legibility on [2011 research published in Journal of Vision](https://jov.arvojournals.org/article.aspx?articleid=2191906#88123043). The study found a correlation between the degree of visual angle and reading speed. If we make the assumption that people are able to read faster when text is more legible, we are able to draw a correlation between visual arch and legibility.

## Typefaces

Coinbase Sans is a type set

- **Coinbase Display** is optimized for large display text where characters are the most expressive. It's ideal for headings where only a few words are displayed together. We recommend using it for text that is above 24pts.

- **Coinbase Sans** is best suited for medium to longform content. It's ideal for most copy and longer headings. We recommend using it for text that is between 14-24pts.

- **Coinbase Text** is our small text focused typeface. It's data-rich views like dashboards or trading experiences. We recommend using it for text that is below 14pts.

- **Coinbase Mono** is a technical typeface for showcasing code or wallet addresses.

### Visual angle overview

Visual angle is a way for us to understand and quantify how much space an object takes up in the cone of our vision. In the diagram below, we are able to draw a triangle between the eye and piece of type and calculate the visual angle. Because a vast majority our interface text is lowercase, we measure the height of text with x-height. For vision quality, we can assume that a large majority of our users have 20/20 vision either naturally or with corrective lenses. So for vision quality, we consider the quality of the display by the density of pixels.

### Incorporating research

From the 2011 study research, this chart shows the reading speed using the RSVP method. Using this chart, we can see a sharp increase in people's ability to read faster as the visual angle increases and slowly diminishes as the arch increases. With this in mind, we concentrated our type sizing in the zone between .2° and .3° to maintain minimum and maximum thresholds.

When type sizing falls below .2° visual arch, for some styles we modify the style to be all caps and for others we maintain .2° at lower scales meaning the type style will not go below its minimum size.
