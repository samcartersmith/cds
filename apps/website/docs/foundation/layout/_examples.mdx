We have quite a few layout components in our library, but how do you know when to use which component? We're going to take a look at some real-world layouts and show you how to compose them.

In this guide we'll be building a simplified version of the [NFT Shop](https://nft.coinbase.com/shop) page with CDS layout components. There are actually many layouts in just this one screen, so we're going to break them down into sections and show you how to build them.

### Layout Components

These are the components we'll be using:

| Component                     | Description                                                                                                           |
| ----------------------------- | --------------------------------------------------------------------------------------------------------------------- |
| [Box](/components/box)        | A flexbox container that supports many style properties                                                               |
| [HStack](/components/h-stack) | A specialized Box that lays out its children in a row. Optionally can add a gap between children                      |
| [VStack](/components/v-stack) | A specialized Box that lays out its children in a column. Optionally can add a gap between children                   |
| [Grid](/components/grid)      | A specialized Box that lays out its children in columns of specified sizes. Optionally can add a gap between children |

### Terminology

It also helps to have a common language when we're discussing layouts. These are some terms we'll use quite often:

| Terminology     | Definition                                                                                                                                                                  |
| --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Tracks          | Generic term for both columns and rows. See [CSS spec](https://www.w3.org/TR/css-grid-1/#grid-track-concept)                                                                |
| Grid Lines      | The lines created by CSS grid that delineate columns and rows, including around the entire grid itself. See [CSS spec](https://www.w3.org/TR/css-grid-1/#grid-line-concept) |
| Explicit Layout | You decide how many tracks are in a row/column (optionally, depending on breakpoint).                                                                                       |
| Implicit Layout | The available space determines how many tracks will be laid in a row/column.                                                                                                |

## Sidebar + Main Content

Let's first look at the top level layout that places the sidebar next to the main content. There are two ways we can do this, depending on what the design specifications are.

### Proportionate Columns

Say design wants the main content to be 4 times as large as the sidebar, we can use [HStack](/components/h-stack) and [VStack](/components/v-stack)'s `flexGrow` property to create a proportionate relationship between the sidebar and the main content.

:::note
On smaller screen sizes, flexbox will not always dedicate 4 times the space to the main content when there isn't enough space available. But that is one of the benefits in a flexible system - you don't have to be so explicit! You can simply tell the browser what is ideal, and it will handle the layout. If you _do_ have strict requirements, you can use the `minWidth` prop on the main content or pass a configuration object to `responsiveConfig` with different values for `flexGrow` depending on the breakpoint.
:::

```jsx live
// Wrap everything in a Stack component
// To make it responsive, add flexWrap and it'll collapse to a column when the content can't fit within the available space
<HStack gap={4} flexWrap="wrap">
  <VStack>
    {/* Sidebar starts here */}
    <TextLabel1 as="p" spacingBottom={3}>
      Search
    </TextLabel1>
    <SearchInput
      value=""
      compact
      onChangeText={(val) => console.log(val)}
      placeholder="Search NFTs"
    />
  </VStack>
  {/* Make sure you add flexGrow to the content that you want to be n times larger than the other */}
  <VStack flexGrow={4}>
    <TextHeadline as="h2">Main content</TextHeadline>
  </VStack>
</HStack>
```

### Fixed Width + Free Space

If design knows how wide the sidebar should be (let's say `240px`), you can use Grid to specify the column widths. For the main content, we can use the `1fr` dimension which just tells the browser to take up the remaining free space.

Unfortunately, this layout is not responsive, though. However, you can use the values destructured from [useBreakpoints](/hooks/useBreakpoints) to conditionally render different values for `templateColumns` or different component trees depending on the breakpoint.

:::tip
If you are going to use useBreakpoints, make sure you wrap your component in [BrowserOnly](/components/BrowserOnly), since it relies on the `window` object which is not present in server side environments.
:::

```jsx live
// Wrap everything in a Grid component
<Grid gap={4} templateColumns="240px 1fr">
  <VStack>
    {/* Sidebar starts here */}
    <TextLabel1 as="p" spacingBottom={3}>
      Search
    </TextLabel1>
    <SearchInput
      value=""
      compact
      onChangeText={(val) => console.log(val)}
      placeholder="Search NFTs"
    />
  </VStack>
  <TextHeadline as="h2">Main content</TextHeadline>
</Grid>
```

## NFT Grid

Great! We have a top level layout, now let's build out this NFT grid that is the main content. We can use the [Grid](/components/grid) component to make a couple different responsive layouts.

### Explicit Columns

Say design wants only a certain number of NFT's to be shown on each row, depending on which device we're using. We can leverage the "smart" `columns` prop, which will create equal columns of a specified size. And then we can pass a config to `responsiveConfig` with the number of columns we want at each breakpoint.

The only gotcha with this method is that we need to make sure we can actually fit the number of NFT's we want at all these breakpoints. Say each NFT has a `minWidth` of `400px`, at some screen sizes, it may break the bounds of the Grid. In that case, if you're going to reach for explicitly declared columns, make sure you don't have a fixed width on the Grid's children, instead allow them to size themselves based on the available space.

```jsx live
function ExplicitGrid() {
  const responsiveNFTGridConfig = useMemo(() => ({
    phone: {
      columns: 1,
      gap: 1,
    },
    tablet: {
      columns: 2,
      gap: 2,
    },
  }));

  return (
    <Grid columns={3} gap={3} responsiveConfig={responsiveNFTGridConfig}>
      {nftData.map((props, idx) => (
        <NFTTile key={idx} {...props} />
      ))}
    </Grid>
  );
}
```

### Implied Columns

Alternatively, if you don't care how many NFT's are in a row and you just want to fit as many as possiblel; you can use the implicit properties of Grid.

Make sure you pass values for `columnMin` and optionally `columnMax`, which will help the Grid know how many children it can fit in a row.

:::tip
Inspect the Grid component with your dev tools and press the `grid` tag next to the DOM element to see how the browser is actually rendering the grid. You'll notice there are lines outlining the entire grid, as well as lines between each column. Resize the screen and watch how the browser recalculates the track sizes.
:::

```jsx live
function ExplicitGrid() {
  const responsiveNFTGridConfig = useMemo(() => ({
    phone: {
      columns: 1,
      gap: 1,
    },
    tablet: {
      columns: 2,
      gap: 2,
    },
  }));

  return (
    <Grid columnMin="180px" columnMax="200px" gap={3}>
      {nftData.map((props, idx) => (
        <NFTTile key={idx} {...props} />
      ))}
    </Grid>
  );
}
```

## Other Layouts

:::tip
Grid is not always the best option for layouts, though. When Grid creates columns, whatever the dimensions are for each column in the first row will carry over to subsequent rows.
:::

If you want to layout a bunch of content that wraps to other lines and the width of the content is variable, you'll want to reach for HStack instead.

```jsx live
function TagExample() {
  return (
    <HStack flexWrap="wrap" gap={0.5}>
      {nftTagNames.map((nft) => (
        <HStack spacingBottom={0.5}>
          <Tag colorScheme="blue" key={nft}>
            {nft}
          </Tag>
        </HStack>
      ))}
    </HStack>
  );
}
```
