## Overview

Illustration plays a significant role at Coinbase in conveying complex ideas in a simple but powerful way. These visuals can take shape and form to help elevate experiences, providing education, and echoing clarity when needed. Illustrations are custom created per experience by the Illustration Team and can be found in our Illustration Component on CDS and Figma.

## Using illustration

When thinking about utilizing an illustration, it typically aims to be one or more of the following goals:

- **Making complex ideas more understandable** by allowing the illustration to be universally understood will help better the cognitive load for all.
- **Function over aesthetics**, as product illustrations play more of a user experience role rather than being ornamental.
- **Being considerate of the user's emotional state** by using illustrations to convey particular messaging and tone to help get specific tasks done (e.g. card declined, missing account details, transaction completed, etc.).
- **Help tell stories and thoughtfully convey ideas**, although not to be used as decoration or without consideration.
- **Focus attention** by creating a visual hierarchy.
- **Represent our brand tone / voice;** as product illustrations serve to be more functional, there is an opportunity to showcase our personality and voice when necessary.

## Types of illustrations at Coinbase

Coinbase illustrations come in various types and forms, and it's important to note their differences to empower your work.

### Pictograms

<Image
  light="/img/foundation/illustration/pictograms_light.png"
  dark="/img/foundation/illustration/pictograms_light.png"
/>

Pictograms can be best described as an elevated product icon, as it still carries the same thinking. However, a pictogram is visually different from a product icon with its larger dimensions, multiple uses of colors, and increased detail. Pictograms are most commonly used within a composition of other pictograms (e.g. in a list of categories within an NFT marketplace). Pictograms don't include any decoration elements, backgrounds, or motion, as they should be simple whilst conveying a concept's literal and singular expression.

**Sizes**

- 48 x 48px
- 64 x 64px
- 96 x 96px

## Illustration treatments

**Icons and illustrations should not be together on a single screen;** choose either the illustration or the icon.

## Accessibility

### Illustrations are always paired with text

Illustrations should never work alone, especially in the lens of accessibility; we always pair illustrations with text. When thinking of users with visual impairments and screen readers, it's key to add thoughtful text that matches the graphic.

## Use Cases

Illustrations live and breathe in many surface areas across Coinbase. We've provided guidance on how they should be considered within these components and experiences.

### Onboarding

Onboarding helps users set up their accounts on Coinbase.

  <Item

      dark: '/img/foundation/illustration/onboarding_light.png',

**Tips**

- These flows entail a lot of task-driven activities. There isn't much room to be delightful, as we want the user to get the task at hand done (e.g. adding their payment details, providing evidence of identity etc.).
- There is room to be delightful at the start and end of the onboarding flow.
- We don’t encourage any use of abstract UI, especially if users haven’t experienced the product. We recommend being as literal as possible when using illustrations.
- Onboarding experiences tend to have a larger surface area, so hero illustrations are a common use of illustrations because of the larger playing field.

**Types of illustrations used**

- Hero (240 x 240px)

### New user experience (NUX)

As we continually grow our features and products, using NUX becomes a common use case to introduce these moments and convert into users.

<Collection>
  <Item

      dark: '/img/foundation/illustration/nux_light.png',

**Tips**

- These experiences are opportunities to be very narrative-driven. The designer and illustrator should collaboratively think about the start and end of a flow and what expectations should come from it: exciting, informative, or a moment of achievement?
- We don't want users to feel overwhelmed, especially when learning a new feature for the first time. Instead, we keep illustrations bite-sized, focused on the task at hand, yet still be exciting.
- These flows should be 2-4 screens, as users retention to remember content fits within those realms.

**Types of illustrations used**

- Hero (240 x 240px)
- Spot (all sizes)
- Pictograms (48 x 48px for cards)

### Modal Dialogs

Illustrations used in modal dialogs tie in with critical information, and exclusively focus on a singular task for the user to complete. Illustrations become more of a functional element to this experience.

- Use pictograms when trying to be non-distracting and direct. Good examples are when card information is rejected and transaction approved.
- Use hero illustrations when given a moment of delight or education. Good examples are the feature benefits for a new product or completing a workflow to end the user on a high note.
- Be considerate on the emotional state of the user when picking illustrations from our library.

**Types of illustrations used:**

- Hero (200 x 200px)
- Pictograms (all sizes)

## Using the Illustration Library

We have an extensive library free to use on Figma and the [CDS site](https://cds.cbhq.net/components/hero-square). These illustrations are reusable; however, we always recommend sending a new illustration request to review the work with you. Each experience and copy is different, and we serve illustrations the same way.

If you do plan to reuse an illustration from the library, please use this framework as a way to measure the success:

- **Does this illustration match the flow with the other illustrations you are using?** As these should feel like a cohesive family, especially if this is an onboarding or carousel experience.
- **Does the illustration match the copy?** We've user tested with our users and found that the connection between visual and text should be perfectly aligned. If not, it's considered to be a broken experience.
- **What is the emotional state of the user when seeing this illustration?** Should the user be focused on a task/be educational or informative/user feel excited? These are all things you should be considering when picking an illustration from our library and ensuring it's the right one to convey a particular emotion to the user journey.

## Requesting an illustration

You can make an illustration request here: go/illustrationrequest. Please make sure you're requesting support at least 2 weeks prior to the due date. You can also check our readily available Illustration Library on CDS or Figma, as we have an extensive library of illustrations.

We do not allow for external contributions, so please make a request or ask in the #ask-illustration for further assistance.

### Approvals and finalization

The Illustration Team will make the final call on quality to ensure consistency with the system. Once this is done, the Illustration Team can add the illustration into CDS.
