Internally the `ThemeProvider` component includes a div element which is used to attach CSS variables that are needed for palette overrides. If the div added by ThemeProvider is not desired, you can use this hook and attach the returned className and style prop to the element of your choice.

:::caution

If you opt to use the `useThemeProviderStyles` hook, you will need to wrap the root of your application with the `FramerMotionProvider` from `@cbhq/cds-web/animation/FramerMotionProvider` to enable animations and `ContextOnlyThemeProvider` to sync palette, spectrum and scale with React context.

:::

```jsx
import { FramerMotionProvider } from '@cbhq/cds-web/animation/FramerMotionProvider';
import {
  useThemeProviderStyles,
  ContextOnlyThemeProvider,
} from '@cbhq/cds-web/system/useThemeProviderStyles';

function App() {
  const { className, style } = useThemeProviderStyles();
  return (
    <FramerMotionProvider>
      <ContextOnlyThemeProvider>
        <main className={className} style={style}>
          <AppRoutes />
        </main>
      </ContextOnlyThemeProvider>
    </FramerMotionProvider>
  );
}
```

  );
}
