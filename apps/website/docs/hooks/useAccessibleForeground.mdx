> [@cbhq/cds-(web|mobile)/color/useAccessibleForeground](https://github.cbhq.net/frontend/cds/blob/master/packages/web/color/useAccessibleForeground.ts)

By default does the following checks on a given color:

1. Check if color meets contrast criteria.
2. If pass, return color passed in.
3. If fail, check saturation.
4. If gray color return `gray100`.
5. If non gray color return closest CDS spectrum alias (i.e. `orange60`).

## Examples

Color can be any valid color (hex, rgb, rgba) including

```jsx live
function Example() {
  const foreground = useAccessibleForeground({ color: assets.dai.color, usage: 'normalText' });
  return (
    <ExampleWithThemeToggles>
      <VStack gap={2}>
        <TextTitle2
          as="p"
          dangerouslySetColor={foreground}
        >{`Accessible foreground: ${foreground}`}</TextTitle2>
        <TextTitle2
          as="p"
          dangerouslySetColor={assets.dai.color}
        >{`Original color: ${assets.dai.color}`}</TextTitle2>
      </VStack>
    </ExampleWithThemeToggles>
  );
}
```

<h2 style={{ display: 'inline-block' }}>Using background & color: auto</h2> <Badge variant="danger" />

- Only use the `background` param if you absolutely have to. For example, if you are using an asset color to power the background color.
- The `auto` prop will return white or black depending on which one has the highest contrast ratio based on the `background` color.

```jsx live
function Example() {
  return (
    <ExampleWithThemeToggles>
      {assetColors.map((assetColor) => (
        <Box height={50} spacing={2} dangerouslySetBackground={assetColor}>
          <TextBody
            as="p"
            dangerouslySetColor={useAccessibleForeground({
              background: assetColor,
              color: 'auto',
              usage: 'normalText',
            })}
          >
            {assetColor}
          </TextBody>
        </Box>
      ))}
    </ExampleWithThemeToggles>
  );
}
```

## Params

### background <Badge variant="danger" />

- Optional param. Will fallback to use the `background` color from palette when undefined.
- Only use the `background` param if you absolutely have to - for example, if you are using an asset color to power the background color.
- Accepts any valid color (hex, rgb, rgba).

```
rgb(255, 255, 255)
rgba(255, 255, 255, 0.4)
#ffeeaa
```

### color

- Any valid color (hex, rgb, rgba) or `auto`.
- The `auto` prop will return white or black depending on which one has the highest contrast ratio based on the `background` color.

```
rgb(255, 255, 255)
rgba(255, 255, 255, 0.4)
#ffeeaa
```

### usage

- `normalText`
- `largeText`
- `graphic`
