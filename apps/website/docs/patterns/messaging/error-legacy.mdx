An error state is a screen that is shown when things go wrong. It is an example of a situation where
the user is getting something other than their desired state. Every error, regardless of cause, becomes
a point of friction for your users and blocks them from moving forward in their experience.

## General guidelines

Error messages range in severity and differ by origination. The way we communicate to the user through the content and the visual treatment of the error can help users better understand why the error is occurring and what to do next.

<Link to="https://coda.io/d/CDS-Team_dFaC-pktuzN/Messaging-States-Research_suRy8">
  Read Error Messaging research →
</Link>

### Look

- Do not solely use color to signify error. Add an error message or an illustration (sparingly).
  - Use the color red sparingly
- An error should almost always have copy and, occasionally, an illustration.
  - Illustrations can reduce confusion and frustration, putting a user at ease when things aren't working as desired. Use error state illustrations sparingly but with intent. Consider how many error state illustrations appear on a singular screen. These illustrations set the tone that something is wrong.
  - Search for these illustrations in our search with the name of the error (e.g. "404", "500", etc.) or "error" in the search bar. We have an ‘error’ icon which is in its name.
- The larger the component, the more visual noise it creates on a page.

### Sound

- Error messages should be clear, concise, and use consistent vocabulary.
- Error messages should be written according to local standards and guidelines.
- Use sentence case capitalization.
- If we know what action the user needs to resolve the problem detailed in the error, calls to action should be specific to that action (e.g. use ‘Refresh the page’ rather than ‘Try Again’).

<Link to="https://docs.google.com/document/d/1QnxOOjhqUb8ENrGt3hQTTDCDxnCPRyEfkO-U-ftm4uc/edit#heading=h.xo750yt3a4gs">
  Coinbase Content Design Guide →
</Link>

### Feel

- Error messages should focus a user’s attention on the problem at hand and how it can be resolved.
- Use animation consistently to set clear expectations for the user.
- Always consider the user's emotional and psychological state. Seeing an error message, particularly certain colors and symbols/icons, can be concerning as visual alerts set the tone for how a user perceives an incident.
- Only use dramatic animation for urgent messages.
- Use hint motion sparingly and with intent. It should be applied to components or surfaces that need immediate attention. Adding or combining various hint systems in a single view can control the severity of the tone. Overstacking can cause a negative experience.
- Origination of the error can be clarified with both color and writing. System errors can leverage red to communicate errors originating from Coinbase. Where applicable, refer to us as Coinbase rather than “we”.
