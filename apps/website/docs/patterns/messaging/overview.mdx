# Messaging

A message in the UI is one of the most direct forms of communication our product has with its users. It can elicit strong emotions in a user, and more importantly, hinder or help in their use of our products. Poorly executed messages can lead to escalations with customer success that can be resource intensive and hurt brand equity.

## Guiding Principles

All messages should:

- Be clear
- Be helpful
- Instill confidence
- Offer security

All elements (component, color, motion, illustration, and copy) work in tandem to establish these outcomes.

## Classification system

This classification system is the foundation of the decision trees that will help you determine which component to use for your message in each messaging state. They are the fundamental divisions for why a message is appearing, what the message is intending to convey, and how it should be incorporated into the user flow and experience.

- Passive (no action needed) → Active (action required)
- Mild (non-critical) → Urgent (critical)
- Ignorable → In need of immediate attention
- Dismissible → Persistent
- Task triggered → system generated
- Non-financial implications → Financial implications
