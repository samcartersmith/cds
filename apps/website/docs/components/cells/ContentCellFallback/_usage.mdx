Supports the same structure and similar props as `ContentCell`, but instead of accepting content, it accepts a boolean. This boolean determines whether to show a fallback shimmer in the desired location.

```jsx live
<ContentCellFallback title subtitle description meta media="avatar" />
```

:::tip Rectangular fallback widths

This component contains rectangular fallbacks, the widths for which are randomized within a predetermined threshold by default (e.g. to add some variety when mulitple fallbacks are presented together). If this behavior is undesirable (e.g. in server-side rendered web apps), randomization can be disabled with the `disableRandomRectWidth` prop.

<br />

Alternatively, you may create a variant with different rectangle widths by setting a number value on the `rectWidthVariant` prop. Variants map to a predetermined set of width values, which are cycled through repeatedly when the set is exhausted. Therefore, it's still possible to achieve some variety in your fallbacks, but in a deterministic manner (i.e. safe for server-side rendering). Here's an example:

<br />

```jsx live noInline
const items = [{}, {}, {}];

function RenderContentCellFallbacksInList() {
  return (
    <VStack gap={1}>
      {items.map((item, i) => (
        <ContentCellFallback title subtitle description meta media="avatar" rectWidthVariant={i} />
      ))}
    </VStack>
  );
}

render(<RenderContentCellFallbacksInList />);
```

:::

Here is an example of how to use dangerouslySetIterations.

```jsx noInline
function DangerouslySetIterations() {
  return <ContentCellFallback title dangerouslySetIterations={30} />;
}
```

:::danger DangerouslySetIterations

On mobile, ContentCellFallback loops for 10 times and stops animating after that. We do
this for performance reasons, and anything that loads for more than 10s should
not go to production anyways.

<br />

On web, it doesn't have this restriction. With that said, if something is loading for more than 10s, you should seriously re-consider the architecture.

<br />

If you truly need the animation to exceed more than 10s on mobile, here is an escape hatch.

<br />
:::
