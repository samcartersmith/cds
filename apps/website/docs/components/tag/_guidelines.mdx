A 1 to 2 word text/string component used for describing, categorizing or classifying content, displaying metadata and/or highlighting sentiment.

## Overview

Tags draw attention to new or important content. Specifically, they (1) draw attention to something that might otherwise be missed or (2) provide commentary or metadata about a larger component.

#### Tag principles

1. Intentional
2. Concise and Scannable
3. Trustworthy/Helpful decision-making
4. Not a button

Tags do not filter content like tags or tabs. Though they are made up of a text string, they are not hyperlinks like buttons. Tags can be accompanied by Tooltips to display overflow/truncated strings or to provide more context/definition of the tag string.

### Content Design Guidelines

#### Informational and Promotional Tags

Informational tags are used to note a characteristic or state of an object. The relationship between the tag and the other UI component should be easy to interpret for a user. Informational tags are often used to describe the state or status of an element. It is expected that the status or state of the element will change. But the various types of states an element can have are uniform and the text will change as the status or state changes.
Promotional tags are for editorial, ephemeral communication. Terms like “New”, “Recommended”, “Beta”, “Hot”, “Trending”, fall into this Promotional tag category. They are placed in close proximity to another larger UI component, or sometimes “badged” on top of a UI element. Promotional tags are an ephemeral UI that “disappears” or “expires” once the user has interacted with the element or once the tag’s text is no longer true. For example, a product will not stay “Beta” forever, Trends have a short lifespan, things that are “Hot” eventually cool off.

> Note: For scannability and clarity, only one tag should be displayed at a time. However, if the tags are used to call out product features, multiple tags can be used. Promotional tags should be used sparingly and singularly. The usage of Promotional tags should be evaluated holistically. A screen full of Promotional tags can become overwhelming, deteriorating the value and trust of the tag.

### Tag taxonomy

Tag terms should be well-known/understood, part of your audience’s vernacular, tag terms should be part of a larger taxonomy rather than a one-off concept.

Tags must always include text. They cannot be empty or use a single character symbol, eg “$”. Icons should be considered a tool for accessibility and therefore should match the text.

- Tags should be 1-2 words, max
- Tags should not extend or wrap to two lines.
- In normal scale, tags should be sentence case unless using Promotional tags.
- Be consistent across a product with the color, style, and text of a tag.
- Do not use the same text in two different tag types in the same user experience.
- Do not use more than one type of tag with a single UI component.
- When possible, limit the use of both Informational and Promotional tags on the same page and/or in the same table.

### Accessibility

- Text must be clear enough for people to be able to understand immediately without needing to rely only on color.
- Avoid using tags for longer text. Keep tags to two words max. Using color to add meaning only provides a visual indication, which will not be conveyed to users of assistive technologies – such as screen readers.
- Ensure that information denoted by the color is either obvious from the content itself (e.g. the visible text), or is included through alternative means, such as additional text hidden with the .visually-hidden class.
