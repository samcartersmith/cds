# Definition

Tooltip is a floating text label that displays information related to an element.

The content of Tooltip is contextual, useful, and nonessential. The element paired with Tooltip may or may not have a visual indicator like an info-circle on desktop.

# Guidelines

<VStack>

<HStack gap={3}>
  <VStack width="50%">
    <TextHeadline as="p">When to use:</TextHeadline>
    <ul>
      <li>To provide helpful, but non-essential context to a UI element.</li>
    </ul>
  </VStack>
  <VStack width="50%">
    <TextHeadline as="p">When not to use:</TextHeadline>
    <ul>
      <li>
        To display information that is critical to the understanding of an element or feature such
        as an error, instructions for performing a task, or interaction feedback.
      </li>
    </ul>
  </VStack>
</HStack>
<HStack gap={3}>
  <VStack width="50%">
    <TextHeadline as="p">Do:</TextHeadline>
    <ul>
      <li>Describe only the element paired with Tooltip.</li>
      <li>Use succinct and scannable language.</li>
      <li>As much as possible, limit Tooltip's text to a maximum of 60 to 80 characters.</li>
      <li>Use an info-circle icon button when the paired element is text-based.</li>
      <li>Provide a clear accessibility label</li>
    </ul>
  </VStack>
  <VStack width="50%">
    <TextHeadline as="p">Don't:</TextHeadline>
    <ul>
      <li>Describe elements other than the one that triggers Tooltip.</li>
      <li>Use Tooltip to restate text already visible on screen.</li>
      <li>Use a Tooltip on Web / Mobile or RN without a visual indicator.</li>
      <li>
        Use a Tooltip on Web / Mobile or RN for an interactive element (which already has a pressed
        state). Instead, consider another layout / pattern for the information.
      </li>
    </ul>
  </VStack>
</HStack>
</VStack>

# Behavior

<HStack>
  <VStack width="50%">
    <TextHeadline as="p">
      Tooltip appears by requested behavior (user initiated). It typically appears after a small
      delay.
    </TextHeadline>
  </VStack>
</HStack>

<HStack gap={3}>
  <VStack width="50%">
    <TextHeadline as="p">For Web/Desktop:</TextHeadline>
    <ul>
      <li>Initiate when element receives keyboard focus or mouse hovers over it</li>
      <li>Element may or may not have a visual indicator</li>
      <li>Dismiss when escape is pressed or on mouse out</li>
      <li>Scrolling will dismiss Tooltip</li>
    </ul>
  </VStack>
  <VStack width="50%">
    <TextHeadline as="p">For React Native:</TextHeadline>
    <ul>
      <li>Initiate when the element's visual indicator is tapped</li>
      <li>Element must have a visual indicator</li>
      <li>Dismiss when escape is pressed or by tapping outside of element</li>
      <li>Scrolling will dismiss Tooltip</li>
    </ul>
  </VStack>
</HStack>

# Size Variant

<HStack gap={3}>
  <VStack width="50%">
    <TextHeadline as="p">
      There are 2 sizing options that are dependant on the text string length.
    </TextHeadline>
  </VStack>
</HStack>

<HStack gap={3}>
  <VStack width="50%">
    <TextHeadline as="p">Default Responsive</TextHeadline>
    <ul>
      <li>Container hugs content</li>
      <li>Supports short text strings</li>
      <li>Contents are center aligned</li>
    </ul>
  </VStack>
  <VStack width="50%">
    <TextHeadline as="p">Block 260pt</TextHeadline>
    <ul>
      <li>
        When character length expands container to 260pt [approx 32 - 36 characters], Tooltip
        defaults to Fixed Width
      </li>
      <li>Container width never exceeds 260pt and text wraps</li>
      <li>Contents are left-aligned for LTR languages and right-aligned for RTL</li>
    </ul>
  </VStack>
</HStack>

# Accessibility

<ul>
  <li>
    Label the tooltip triggering button with the context of what information users will learn from
    it using the accessibilityLabel prop.
  </li>
  <li>When tooltips are closed ensure focus is returned to the triggering tooltip button.</li>
  <li>Ensure the tooltips receive keyboard focus and can be opened without a mouse.</li>
</ul>
