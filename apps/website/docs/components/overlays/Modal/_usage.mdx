### Get Started

1. Add [PortalProvider](/components/portal-provider) to the root of your app. Import it from `import { PortalProvider } from '@cbhq/cds-web/overlays/PortalProvider';`.
2. Compose `Modal` with `ModalHeader`, `ModalBody`, and `ModalFooter` like below for consistency.

```
<Modal>
	<ModalHeader />
	<ModalBody>
		<AnyContentYouWant />
	</ModalBody>
	<ModalFooter />
</Modal>
```

3. Wire up `visible` and `onRequestClose` to control visibility.

### Default Modal

Modal is using React Portal by default, which will port the node to the end of the `body` tag. You can add `disabledPortal` to disable this behavior.

```jsx live
// import { PortalProvider } from '@cbhq/cds-web/overlays/PortalProvider';

function DefaultModalExample() {
  const [visible, { toggleOn, toggleOff }] = useToggler();
  const [scrolling, { toggle: toggleScrolling }] = useToggler();

  return (
    <ExampleWithThemeToggles showFrontier>
      <PortalProvider>
        <ButtonGroup>
          <Button onPress={toggleOn}>Open Modal</Button>
          <Switch onChange={toggleScrolling} checked={scrolling}>
            Show long content
          </Switch>
        </ButtonGroup>
        <Modal visible={visible} onRequestClose={toggleOff}>
          <ModalHeader title="Basic Modal" />
          <ModalBody>
            <TextBody as="p">{loremIpsum.repeat(scrolling ? 50 : 1)}</TextBody>
          </ModalBody>
          <ModalFooter
            primaryAction={<Button onPress={toggleOff}>Save</Button>}
            secondaryAction={
              <Button variant="secondary" onPress={toggleOff}>
                Cancel
              </Button>
            }
          />
        </Modal>
      </PortalProvider>
    </ExampleWithThemeToggles>
  );
}
```

### Programmatically Control Modal

`PortalProvider` is a convenient way for adding anything to the CDS portal. The CDS portal is what CDS uses to centralize all elements that need a portal and handles the zIndex of those pieces.
You can control Modal visibility programmatically using `openModal()` and `closeModal()` from the `useModal()` hook which consumes `PortalProvider` under the hood.

To **open** the modal, pass your `Modal` component into `openModal()` and pass in the `closeModal` handler to the `onRequestClose` prop to make the modal close properly on pressing the X button.

To **close** the modal, use the `closeModal()` function from `useModal()`.

Multiple Modals? No problem, check the example [here](/hooks/useOverlay#multiple)

```jsx live
// import { PortalProvider } from '@cbhq/cds-web/overlays/PortalProvider';
// import { useModal } from '@cbhq/cds-web/overlays/useModal';

function PortalModalExample() {
  function PortalModal() {
    const { openModal, closeModal } = useModal();

    const handlePress = useCallback(
      () =>
        openModal(
          <Modal visible onRequestClose={closeModal}>
            <ModalHeader title="Basic Modal" />
            <ModalBody>
              <TextBody as="p">{loremIpsum}</TextBody>
            </ModalBody>
            <ModalFooter primaryAction={<Button onPress={closeModal}>Save</Button>} />
          </Modal>,
        ),
      [openModal, closeModal],
    );

    return (
      <Box>
        <Button onPress={handlePress}>Open Modal</Button>
      </Box>
    );
  }

  return (
    <PortalProvider>
      <ExampleWithThemeToggles>
        <PortalModal />
      </ExampleWithThemeToggles>
    </PortalProvider>
  );
}
```
