#### When to use accordions

> Containing content in an accordion can reduce cognitive load, eyeball glaze and make the screen appear less daunting.

- To organize related information.
- To shorten pages and reduce scrolling when content is not crucial to read in full.
- When space is at a premium and long content cannot be displayed all at once
- when people need only a few key pieces of content on a single page. By hiding most of the content, users can spend their time more efficiently focused on the few topics that matter.
- Accordions serve as a mini-IA of the page, allowing users to form a mental model of the information available.

#### When NOT to use accordions

> Forcing people to click on headings one-at-a-time to display full content can be cumbersome, especially if there are many topics on the list that individuals care about. **If people need to open the majority of subtopics to have their questions answered or to get the full story then an accordion is not the way to go.** In this situation, it's better to expose all the content at once using CDS List Cells. It is easier to scroll down the List with all the exposed content, than to have to decide which heading to expand.

Accordions hide content from the user. Not all users will notice accordions or understand how they work. For this reason, you should only use accordions in specific situations and if user research supports it.

- Do not use accordions for navigation. Use [CDS Side Nav](/components/sidebar) or [CDS Tabs](/components/tab-navigation)
- Do not use an accordion for content that all users need to see at once. Use CDS Lists
- Do not use accordions for content if you want all accordion items expanded. Use CDS Lists
- Do not nest accordions within accordions, as it will make content difficult to find, and degrade usability.
- Do not use accordion if you need to expand and collapse a group of components. Use [Collapsible](/components/collapsible).

#### Head Variants: Title, SubTitle

Accordions are responsive to screen sizes and localization. Therefore the number of lines in the header will vary. Test your Accordion designs in various languages and screen sizes to ensure the content does not bloat the header and overwhelm the UI. See [Accordion's Content Guidelines](#content-guidelines) for more guidance on crafting header titles.

The Accordion header is designed for 2-lines max:

- 1-line Title and 1-line SubTitle; or a
- 2-line Title, text wrapping is supported.

### Accessibility

AccordionItems inherit focus states built into ListItems. When an accordion header has keyboard focus, the styling of the accordion container and all its header buttons is changed.

**When any accordion header receives focus:**

- The border encompassing the entire accordion changes color on keyboard tab. The border encompasses both the header text, left icon (if applicable) and zippy/chevron.
- The background color of the accordion header changes on hover distinguishing it from the other accordion buttons that are not focused.

## Content Guidelines

Use accordions to hide content from users; a user may need the information to understand the rest of the page but it should not be critical for understanding. If the content is required by most users, do not put it in an accordion.

Accordions work best for simple content and links. Ideally, an accordion would have about 4-6 items. Accordions increase cognitive load so consider condensing content and displaying in another format if you only have 2 or 3 accordion items.

Do not use accordions as a junk drawer. Although the content is condensed by an accordion, the presence of the accordion still increases the number of interactions on a screen which can burden a user.

  <Image
    src="/img/components/accordion/content_design_example.png"
    options={{
      maxWidth: '50%',
    }}
  />

**Header** → AccordionItem - Collapsed

- Be as brief as possible while remaining descriptive and clear.
- The header can be a question or a title as long as it provides context for what it is in the body copy.
- Headers can be 2 lines, either as a 2 line header or a 1 line header and 1 line sub-header.
- Headers should all be in the same format (eg, all questions or all nouns to be defined)

**Body** → AccordionItem - Expanded

- All body copy should fit within the maximum height of the expanded accordion item. If more space is needed either use a link to a new page with further information or reconsider if an accordion is the correct component to use.
- Body can be split into multiple paragraphs and use sub-headers if needed.
- Body copy in each accordion unit should directly relate to the header next to the zippy.
