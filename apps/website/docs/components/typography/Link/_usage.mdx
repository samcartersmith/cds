## Accessibility

:::tip Accessibility tip

[WCAG 2.0](https://webaim.org/standards/wcag/checklist) has 2 requirements for body text links that are not underlined by default:

<br />

_The link text must have a 3:1 contrast ratio from the surrounding non-link text. The link must present a "non-color designator" (typically the introduction of the underline) on both mouse hover and keyboard focus. These two requirements help ensure that all users can differentiate links from non-link text, even if they have low vision, color deficiency, or have overridden page colors._

:::

### Styling your link

To style a `<Link />` component, simply wrap the `Link` node in the desired `Text` component, using the `as` prop for your usecase (for example, `p` for body text, `h1`/`h2`/`h3` for titles, etc). However, there may be times when a rendering a span is _exactly_ what you want. In those cases, providing a variant prop makes sense.

#### Wrapping your Link in the appropriate Text element

```jsx live
<VStack alignItems="flex-start" gap={3}>
  <TextBody as="p">
    <Link to="https://www.google.com/" openInNewWindow>
      body link
    </Link>
  </TextBody>
  <TextCaption as="span">
    <Link to="https://www.google.com/" openInNewWindow>
      caption link
    </Link>
  </TextCaption>
  <TextLabel1 as="label">
    <Link to="https://www.google.com/" openInNewWindow>
      label1 link
    </Link>
  </TextLabel1>
  <TextLabel2 as="label">
    <Link to="https://www.google.com/" openInNewWindow>
      label2 link
    </Link>
  </TextLabel2>
  <TextLegal as="span">
    <Link to="https://www.google.com/" openInNewWindow>
      legal link
    </Link>
  </TextLegal>
  <TextTitle1 as="h1">
    <Link to="https://www.google.com/" openInNewWindow>
      title1 link
    </Link>
  </TextTitle1>
  <TextTitle2 as="h2">
    <Link to="https://www.google.com/" openInNewWindow>
      title2 link
    </Link>
  </TextTitle2>
  <TextTitle3 as="h3">
    <Link to="https://www.google.com/" openInNewWindow>
      title3 link
    </Link>
  </TextTitle3>
</VStack>
```

#### Using the variant prop

If you find yourself in a situation where you simply need to style a link as a Text component _without_ wrapping it in the semantically appropriate text element, reach for the `variant` prop.

```jsx live
<VStack alignItems="flex-start" gap={3}>
  {['body', 'caption', 'label1', 'label2', 'legal', 'title1', 'title2', 'title3'].map((variant) => (
    <Link variant={variant} to="https://www.google.com/" openInNewWindow>
      {`${variant} link`}
    </Link>
  ))}
</VStack>
```

```jsx live
<Link to="https://www.google.com/" color="negative">
  With color override
</Link>
```

```jsx live
<TextBody as="span">
```

```jsx live

```

:::caution Links nested inside Text components (Mobile only)

As of React Native v0.68.2, nesting a Link inside of a Text component and applying spacing to that Text parent can result in a misalignment of the Link and its hitslop. Until the underlying alignment issues are resolved in React Native, our recommendation is to wrap your composed Text/Link in a Box component and apply any necessary spacing to that wrapper, like so:

<br />

```jsx live
<Box spacingTop={5}>
  <TextBody as="span">
    {'Parent text '}
    <Link to="https://www.google.com">{'link text'}</Link>
  </TextBody>
</Box>
```

:::
