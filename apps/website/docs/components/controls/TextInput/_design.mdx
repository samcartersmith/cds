Text Inputs enable users to type, enter, and/or edit text. Text Field is customizable and has the ability to support various types of objects at the start and end of the input. This component is most commonly used in forms.

Use Text Fields for free form responses or when we can’t predict a user’s answer. Use another component like Select, Date Input, or one of our Group Selection Control components if a user is selecting from a set of options.

1. **Label**: Title / brief description of Input
2. **Input Content**: Content users enter into Input
3. **Helper Text (Optional)**: Assistive message about the Input
4. **Start**: Add an Icon, Asset, Text, Link, or Icon Button to the start of an Input
5. **End**: Add an Icon, Asset, Text, Link, or Icon Button to the start of an Input

## Usage

- Use Text Inputs for free form responses or when we can’t predict a user’s answer.
- Use another component like Select, Date Input, or one of our Group Selection Control components if a user is selecting from a set of options.

## Content Guidelines

- Always include a label.
- Write Input Labels, Placeholder Text, and Helper Text in sentence case.
- Use Helper Text to show hints, formatting, and requirements.
- Use Helper Text instead of Placeholder Text.
- Do not explain what is wrong but rather offer a solution. Error messages should be clear and concise.
- For more info on writing text inputs, check our [Content Design Guide](https://docs.google.com/document/d/1QnxOOjhqUb8ENrGt3hQTTDCDxnCPRyEfkO-U-ftm4uc/edit#heading=h.m7ffxiih1sf1).

## Sizing & Scale

Text Inputs are available in Dense and Normal scales. We also offer a compact option for smaller spaces and dense experiences.

When stacking inputs, use spacing 3.

## Best Practices

### Input Label

Always use a Label to communicate the intention of the Input to a user.

Input Labels can be placed on top of the input or embedded within the input itself. By default, all Inputs will be top aligned. This is to accommodate longer label text, localization, and responsive layouts. Toggle on the compact variant for Embedded Labels. Embedded option for dense, compact spaces.

### Placeholder Text

Please use placeholder text with caution. Because placeholder text disappears as soon as a user enters text, it is not accessible for people with cognitive or visual disabilities. Do not place essential information or instructions related to the input in placeholder text. Instead, use Helper Text.

<Input.Placeholder />

### Helper Text

Helper Text appears below the Text Field and does not disappear. Use Helper Text to provide additional context or instruction about the input. Helper Text is dynamic and can handle sentiment states such as success or error states.

Sentiment states validate data before a form is submitted. Validation, either positive or negative, appears when the user has navigated out of the Input Field.

- **Success message**: Triggered when there is a valid input entry. This message should not disappear.
- **Error message**: Informs the user what has happened and provides guidance on how to resolve the entry. Error messages disappear once the data is corrected and passed as valid.

### Content Alignment

You have the option to align input content to the left or right. However, please use discretion and consistency when doing so. All inputs will be left aligned by default.

We recommend right aligning input content when using the Compact Label variant.
