### Creating a CDS TextArea or Multiline TextInput

```jsx live noInline
const MAX_CHARS = 255;

function RenderNativeTextArea() {
  const [inputText, setInputText] = useState('');

  const onChange = useCallback((e) => {
    setInputText(e.target.value);
  }, []);

  const customNativeTextAreaCSS = useMemo(() => {
    return {
      resize: 'none',
    };
  }, []);

  const charsRemaining = useMemo(
    () => `${MAX_CHARS - inputText.length} characters remaining`,
    [inputText.length],
  );

  return (
    <Box spacing={1}>
      <TextInput
        inputNode={
          <NativeTextArea
            value={inputText}
            onChange={onChange}
            style={customNativeTextAreaCSS}
            rows={7}
            cols={5}
            placeholder="Type something here..."
            maxlength={MAX_CHARS}
          />
        }
        helperText={charsRemaining}
        label="TextArea with character counter"
      />
    </Box>
  );
}

render(<RenderNativeTextArea />);
```

### Accessible NativeTextArea

:::tip Accessibility tip

Like any component system, much of responsibility for building accessible UIs is in your hands as the consumer to properly implement the component composition. We'll do our best to provide sane fallbacks, but here's the biggest gotcha for `NativeTextAreas` you can watch out for.

<br />

#### Always use NativeTextArea with InputLabel or TextInput

Always pair this component with an InputLabel or use it with TextInput. It
is not recommended to use NativeTextArea as a standalone component as it violates accessibility

:::
