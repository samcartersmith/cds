## Usage

Tabs organize and allow navigation between groups of content that are related and at the same level of hierarchy. There needs to be at least two tabs. Limit the number of tabs to six or less.

### Principles

**Accessible**

- Use no more than 6 tabs in a group. For more than that, an accessible solution for access to overflow tabs (see Anatomy).
  **Organized**
- Tabs are used to organize or group information about a single topic into sections.
  **Related**
- Adjacent tabs should have a contextual relationship to each other. Each tab should have equal importance.

## Types

### Primary tabs

- Page level tabs
- Contextual relationship between tabs on the same page.
- Designed to be secondary by nature (sub-navigation).
- Consistent tabs across a set of pages.
- Only one set of primary tabs per page.
- Only one can be selected at a time.

<PrimaryExample />

### Secondary tabs

- Inline tabs
- Contextual relationship between tabs in the same container.
- Designed to be tertiary by nature.
- Can be multiple sets of secondary tabs per page.
- Only one tab can be selected per container at a time.

### Scrolling

On web, if tabs can not fit the width of the container, the tabs that do not fit will go into an overflow menu.

On mobile, if tabs can not fit the width of the container, users can swipe to horizontally scroll through the tab group.

## Content

When writing labels for tabs, follow these guidelines.

- Use clear labels, such as Security or Privacy. Avoid using verbs.
- Avoid using custom labels, such as a person’s name or the name of an asset.
- Labels should be concise (1–2 words).
- Use sentence case capitalization, except for proper nouns like “Coinbase One.”
