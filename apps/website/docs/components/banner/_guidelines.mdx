Banners display an important, concise message that provides actions for users to address or dismiss the banner. They should be displayed at the top of a screen, always below the top navigation bar. Banners persist until they are dismissed by the user or the product/system issue has been resolved.

## Usage

CDS recommends one banner should be displayed at a time

- **Actions**: Banners can contain up to two actions or none at all
- **Dismissal**: Inset banners can be dismissed via action or “x” affordance. Blocking fixed full width banners aren’t dismissible and content scrolls below the Banner.
- **Icons**: Banners must contain an icon that best supports variant type
- **Links**: Banners can support in-line links in the body text

### Banner types

The four types of banners are:

- **Informational**: Inset, passive and informational
- **Promotional**: Inset and full width, promotional
- **Warning**: Inset, warning or error
- **Danger**: Inset and full width, severe warning or error

function InformationalExample() {
  return (
    <ResponsiveExample>
      <VStack gap={2}>
        <Tag colorScheme="blue">Informational</Tag>
        <Banner title="Updates in progress" variant="informational" startIcon="info">
          Additional changes can’t be made at this time.
        </Banner>
        <Banner
          title="Where’s my interest?"
          variant="informational"
          startIcon="info"
          primaryAction={<Link to="/components/banner">Learn more</Link>}
        />
        <Banner

          variant="informational"

          primaryAction={<Link to="/components/banner">Learn more</Link>}
        >
          cbETH is a token that represents ETH staked on Coinbase. The underlying interest you earn

        </Banner>

### When to use

Use a Banner when a general notification or error has occurred for the system or product and a specific task isn’t necessary.

### Cross-Platform

**Web vs React Native:** The only difference is on web where actions move below the body text when the presence of dismiss “x” is needed within the banner.

## Sizing & Scaling

Banners do react to [density](/foundation/scale) changes.

## Content guidelines

- Keep it concise, use short, declarative copy
- Avoid going over over one line for the Headline, three lines for body copy but can be overridden
- Actions should be 1-3 words
