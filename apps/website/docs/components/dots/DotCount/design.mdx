# Dots

**Dots are component adornments,** typically used to communicate a numerical value or indicate the status or identity of a component to the end-user.

## Usage

Dots are used as a descriptor attribute used to help visualize the status of a thing, the # of a thing or the type of a thing. The use cases covered:

- Dot Status Color
- Dot Count
- Dot Symbol

## Principles

Use of Dots should follow the rule of being meaningful, specific and discernible, in that it:

- Does not depend on color. Color alone is not enough to communicate meaning.
- Accompanies text that spells out its meaning in order to avoid making the end-user interpret the Dots intent.
- Displays symbols to communicate an element's identity or attribute

## Dot Count

Dot Count are used to show the amount of a unit of information, usually a communication (such as a message or email). **Dot Counts are ephemeral.** They only appear when there is a count of 1 or more.

- When there is no count, when Dot Count = 0, the Dot is not displayed.
- When count is a single digit Dot Count is shown in a dot/circular shape, but
- When count is double digits, Dot Count shape “pills” to accommodate n
- If count is more than 99, Dot Count truncates to 99+

Positioning: Dot Count can be positioned

- Right or left of the text its associated with or
- "Badged" to the top-right of a component (eg, an icon or avatar) . The Dot Count overlapping the component from the center of the dot

Styling:

- Dot Count comes one size in Normal and Dense, and one color: CDS Red
- When Dot Count is badged to another component a white border is displayed around it

## Accessibility

Dot variants are attributes that are treated as part of a whole and not as separate standalone components. Set up your navigation and aria labels so that the dots are in context to the element it is attributing.

<Image src="/img/components/dots/Example_A11y_01.gif" />

**Tabbing and Tap targets:** Dots do not have a tap target or micro-interactions independent from the component it is attributed to. When a Dot variant appears with a component that has a tap target, the tap target increases to accommodate the component and the Dot. Tapping the Dot or the component should point to the same destination.

**Aria Labels:** When a Dot is visible on your element include a contextual description of the badge's content or purpose
