## 5.0.0

Howdy folks! Here is a list of what's new with this migration. Please use the cds-migrator and go through each following section to ensure your codebase is ready for launch. Happy coding!

Major changes include:

- Splitting ui-mobile-playground into two packages: ui-mobile-playground and ui-mobile-visreg. This reduces the amount of dependencies brought into repos, and easily enables consumers to utilize our same mobile playground UI.
- Components related to select web, dropdown web, tooltip web, popover web, sparkline web and sparkline mobile have been moved into their own standalone packages. Currently this is non-breaking, but we will be removing the instances from cds-web|mobile next quarter so a migration now is recommended.
- RN 71.8 Bump
- React 18 Bump
- Reanimated V3.1.0 Bump
- Update A11y styling for VStack 'ul' components
- [Q1 2023 Deprecations](../../resources/deprecations#q12023-deprecations) were deleted. Use the `cds-migrator` script for `5.0.0` to automatically migrate most deprecations or output migration guidance for manual migrations.

### Happy path upgrade steps

1. From your root NX workspace install the new cds-migrator tool

```shell
yarn add @cbhq/cds-migrator --dev
```

2. From your root NX workspace run the following command:

```tsx
yarn nx generate @cbhq/cds-migrator:migrate 5.0.0
```

3. Run yarn install to capture any updates to package.json

```shell
yarn install
```

and

```shell
lint --fix
```

to automatically sort imports that may have changed during the migration.

4. If you are a mobile app, you will need to re-link native assets. This looks different depending on your setup, but after linking assets you will need to run a fresh native build.

5. Run typecheck to identify any issues not caught with migrator

6. Remove @cbhq/cds-migrator from dev dependencies if you want to avoid having to install unused dependencies. If you removed you will need to re-run install to capture changes in yarn.lock.

### Splitting UI Mobile Playground into two packages

#### Update ui-mobile-playground

If you're consumers of `ui-mobile-playground`, upgrade to 1.0.0.

#### Update `'@cbhq/ui-mobile-playground/visreg';`

This import has been deprecated and those packages have been moved to `@cbhq/ui-mobile-visreg@1.0.0`.

```shell
yarn add @cbhq/ui-mobile-visreg
```

and change your imports from `'@cbhq/ui-mobile-playground/visreg';` to `@cbhq/ui-mobile-visreg`.

#### Update Routes path

The import path for playground routes should be updated from `import { routes } from '@cbhq/cds-mobile/examples/newRoutes';` to `import { routes } from '@cbhq/ui-mobile-playground/routes';`

This branch demos the required path change and works with sparkline in the retail-cds playground: https://github.cbhq.net/consumer/react-native/pull/20330/files#r1115425

### Component Migrations (Select, dropdown, tooltip, popover, sparkline and related components)

As part of an effort to address tech debt in the CDS repo we have opted to migrate a number of problem components out of cds-web|mobile into their own packages (cds-web-overlays, cds-web-visualization, cds-mobile-visualization).

Although these components are not yet removed from cds-web|mobile, we plan to remove them next quarter which will be a breaking change. Our recommendation is to migrate them now using the `cds-migrator` v5 script, which will update import references for these components for you.

Web components moved out of cds-web:

- Dropdown related components
- Select related components
- Popover releated components
- Tooltip related components
- Sparkline related components

Mobile components moved out of cds-mobile:

- Sparkline related components

To see a full list of deprecations for Q22023 navigate to the [deprecations page](https://cds.cbhq.net/resources/deprecations/#q22023-deprecations)

### Handle A11y styling automatically for VStack components that use as="ul"

We removed web api styling (margin-top, margin-bottom, padding-left) for `<Vstack as=ul >` components. The cds-migrator will flag usages of VStack that may be impacted. Please verify that they look as expected.

### Q12023 Deprecations

See [Q12023 Deprecations](../../resources/deprecations#q12023-deprecations) for the full list of deprecations as well as migration guidance. Most deprecations are automatically migratable using `cds-migrator`, however there are a few known issues which require manual review for all migrations:

- Deprecations that require manual migration will be outputted to `cds-migrator-output.md` at the root of the repo
- Dynamic, conditional, and spread props (`...props`) are not automatically migratable, eg: `<Button variant={variant} />` where variant is a dynamic prop. The migrator will output a message to manually migrate these instances. Make sure you run all affected applications to check for runtime errors after migrating.
- Some function migrations were not automatically migratable if a variable was destructured from the function. For example, `const { foo } = bar()` would not be automatically migratable. The migrator will output a message to manually migrate these instances. The inverse is also true, if you assigned a function to a variable and the replacement requires that you destructure a variable, it will be broken, eg: `const { isMobile } = useBreakpoints()` which replaces `const isMobile = useIsMobile()`.
- There is a known issue with the migration script for components that were deprecated and migrated to alternatives where it will sometimes throw an error saying ` >  NX   Attempted to get information from a node that was removed or forgotten.` this is a known issue, and is often seen after a successful migration. We have a fix in the works, so please ignore this error for now.
