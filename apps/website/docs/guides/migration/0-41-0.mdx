## 0.41.0

> [Changelog](/changelog/web#0401-4222022-1149-am-pst)

### PopoverMenu

<Banner
  icon="error"
  title="PopoverMenu is being replaced by Dropdown"
  description="PopoverMenu is being deprecated in favor of Dropdown. We strongly recommend that you migrate instances of PopoverMenu to Dropdown, however migration is not required in order to upgrade to later versions of CDS web."
/>

CDS completely reworked PopoverMenu into an entirely new component called Dropdown, which introduces a new API and paradigms which make it more reusable. Because of this work, migration is not 1:1, so please review the following breaking changes carefully.

The biggest change is that Dropdown does not have controlled visibility. Rather, you can use methods exposed on a Dropdown ref to programmatically control visibility (openMenu and closeMenu, the same names as the visibility handlers on PopoverMenu).

<TextTitle4 as="h4">Removed Props</TextTitle4>

```ts
/**
 * Whether Menu is mounted or dismounted
 * @default false
 */
visible: boolean;
/** Callback that mounts the Menu */
openMenu: NoopFn;
/** Callback that unMounts the Menu */
closeMenu: NoopFn;
/**
 * Enable when PopoverMenu is used with a SearchInput as the trigger. Applies
 * relevant focus behaviors and keyboard events
 * @default false
 */
searchEnabled?: boolean;
```

Dropdown does not yet support a SearchInput or TextInput as the subject. This will be tackled in a separate effort to enable Typeahead functionality in Dropdown compositions.

<TextTitle4 as="h4">New Props</TextTitle4>

```ts
/** Callback that fires when Dropdown is opened */
onOpenMenu?: NoopFn;
/** Callback that fires when Dropdown is closed */
onCloseMenu?: NoopFn;
/** Callback that fires when Dropdown or trigger are blurred */  /** Display a Modal on mobile web instead of a relatively positioned Dropdown */
enableMobileModal?: boolean;
/** Prevent menu from closing when an option is selected */
disableCloseOnOptionChange?: boolean;
/**
 * Display an overlay over all content below the Popover menu
 * @default false
 */
showOverlay?: boolean;
```

<TextTitle4 as="h4">Renamed Props</TextTitle4>

```diff
  /**
   * Makes the Popover Subject fill the width of the parent container
   * @default false
   */
- flush?: booean;
  block?: boolean;
  /** Override content positioning defaults */
- popoverPositionConfig?: PopoverContentPositionConfig;
  contentPosition?: PopoverContentPositionConfig;
```

<TextTitle4 as="h4">Impacted Components</TextTitle4>
<TextHeadline as="p">Switcher</TextHeadline>

Switcher was removed, as it was a one-off components created for Identity to build the App and User Switcher recipes. All Switcher functionality exists in Dropdown, so the Switcher component has been deleted, and the recipes were updated to use Dropdown.

Any implementations using Switcher will need to migrate to the new recipe(s):

- [App Switcher Recipe](/recipes/app-switcher)
- [User Switcher Recipe](/recipes/user-switcher)

This diff is for illustrative purposes, please refer to the respective recipes for each Switcher for the actual implementations.

```diff

-     toggleVisibility.toggle();

-     toggleVisibility.toggleOff();

      placement: 'bottom',
      gap: 1,

    return (
```

<TextHeadline as="p">SidebarMoreMenu</TextHeadline>

Visibility handlers were removed from underlying Dropdown component. There are now two new handlers added to SidebarMoreMenu, `onOpenMenu` and `onCloseMenu` which are fired when the menu opens and closes, respectively.

```diff

  const [collapsed, setCompact] = useState(false);
  const [activeIndex, setActiveIndex] = useState(0);
- const [moreMenuVisible, toggleMoreMenuVisibility] = useToggler(false);

    { title: 'DeFi', icon: 'defi' },

  const moreMenuOptions = [
    {
      value: 'earn',
      label: 'Earn',
      icon: 'earn',
    },
    {
      value: 'borrow',
      label: 'Borrow',
      icon: 'cash',
    },
    {
      value: 'defi',
      label: 'DeFi',
```

<TextTitle4 as="h4">Migration Examples</TextTitle4>

This is a very basic example, but illustrates how you might migrate from using a PopoverMenu to a Dropdown.

```diff
function Default() {
  const [value, setValue] = useState('');
- const [visible, togglePopoverMenuVisibility] = useToggler(false)
  const options = ['Option 1', 'Option 2', 'Option 3', 'Option 4', 'Option 5', 'Option 6'];
  const handleMenuValueChange = useCallback(
    (newValue) => {
      setValue(newValue);
    },
    [setValue],
  );
  return (
+   <Dropdown
-   <PopoverMenu
      width={200}
      value={value}
      onChange={handleMenuValueChange}
-     openMenu={togglePopoverMenuVisibility.toggleOn}
-     closeMenu={togglePopoverMenuVisibility.toggleOff}
-     visible={visible}
    >
-     <PopoverTrigger>
        <IconButton transparent name="more" variant="secondary" />
-     </PopoverTrigger>
      {options.map((option) => (
        <SelectOption
          value={option}
          title={option}
          description="Description"
-         selected={value === option}
        />
      ))}
    </PopoverMenu>
  );
}
```
