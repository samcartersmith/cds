Howdy folks! Here is a list of what's new with this migration. Please use the `cds-migrator` and go through each following section to ensure your codebase is ready for launch. Happy coding!

:::warning
If you upgrade to `@cbhq/cds-web` or `@cbhq/cds-mobile` version `5.10.0` or higher you will also need to upgrade the `@cbhq/cds-illustrations` package to version `2.1.0` or higher.
:::

Major changes include:

- Splitting ui-mobile-playground into two packages: ui-mobile-playground and ui-mobile-visreg. This reduces the amount of dependencies brought into repos, and easily enables consumers to utilize our same mobile playground UI.
- RN 71.8 Bump
- React 18 Bump
- Reanimated V3.1.0 Bump
- Update A11y styling for VStack 'ul' components
- [v5 Deprecations](/resources/deprecations/v5.0.0) were deleted. Use the `cds-migrator` script for `5.0.0` to automatically migrate most deprecations or output migration guidance for manual migrations.
- Sparkline components were moved into their own packages (`@cbhq/cds-web-visualization` and `@cbhq/cds-mobile-visualization`). Migration is optional, but will be required in CDS 6.

This release also removed some illustration dimensions. The `cds-migrator` will update all instances to the nearest smaller supported value. However, this will require manual review, as it could result in visual layout shifts.

You can optionally use the `scaleMultiplier` prop to scale the illustration back to the original size. However, please note that this goes against our most recent design guidance.

## Splitting UI Mobile Playground into two packages

### Update ui-mobile-playground

If you're consumers of `ui-mobile-playground`, upgrade to 1.0.0.

### Update `'@cbhq/ui-mobile-playground/visreg';`

This import has been deprecated and those packages have been moved to `@cbhq/ui-mobile-visreg@1.0.0`.

```shell
yarn add @cbhq/ui-mobile-visreg
```

and change your imports from `'@cbhq/ui-mobile-playground/visreg';` to `@cbhq/ui-mobile-visreg`.

### Update Routes path

The import path for playground routes should be updated from `import { routes } from '@cbhq/cds-mobile/examples/newRoutes';` to `import { routes } from '@cbhq/ui-mobile-playground/routes';`

This branch demos the required path change and works with sparkline in the retail-cds playground: https://github.cbhq.net/consumer/react-native/pull/20330/files#r1115425

## New Packages

Complex components have been moved into their own standalone packages. Migration is optional in v5 but will be required in v6. Use the `cds-migrator` to automatically migrate instances and update package dependencies with `yarn nx generate @cbhq/cds-migrator:migrate package-decomp`.

- `cds-web-visualization`: Sparkline components
- `cds-mobile-visualization`: Sparkline components

### Web Overlays Strategy Change

We previously isolated some overlay components out of the `@cbhq/cds-web` package into their own `@cbhq/cds-web-overlays` package, however we shifted our strategy and will be reintegrating these components back into the core web package. These migrations were optional in v5, however if you already migrated you can use the cds-migrator to migrate instances back to their original locations. You will need to manually remove `@cbhq/cds-web-overlays` as a dependency for your project after running the script.

For more information on this decision visit [go/web-overlays-strategy](http://go/web-overlays-strategy)

```shell
yarn nx generate @cbhq/cds-migrator:migrate revert-decomp
```

Affected components in `@cbhq/cds-web-overlays`:

- Dropdown, Select, Popover, Tooltip, SidebarMoreMenu, and SelectChip

## Update Major dependencies

We now support these major versions and have them listed as peer dependencies. Please upgrade to these versions prior to bumping to 5.0.0.

- RN 71.3 Bump
- React 18 Bump
- Reanimated V3 Bump

## Handle A11y styling automatically for VStack components that use as="ul"

We removed web api styling (margin-top, margin-bottom, padding-left) for `<Vstack as=ul >` components. The `cds-migrator` will flag usages of VStack that may be impacted. Please verify that they look as expected.
