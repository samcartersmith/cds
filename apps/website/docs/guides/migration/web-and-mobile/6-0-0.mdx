### In this major release we eliminated a lot of tech debt, realigned our component API's to newly accepted conventions, enabled Frontier styling globally, exited alpha components, and improved pressable typing. Almost all breaking changes are automatically migratable with the `cds-migrator` (see [Using the Migrator](#using-the-migrator)).

CDS 6 core packages (`common`, `web`, and `mobile`) are only compatible with the following major package versions (or higher):

- `@cbhq/cds-web-visualization` v1.0.0
- `@cbhq/cds-mobile-visualization` v1.0.0
- `@cbhq/cds-illustrations` v2.1.0
- `@cbhq/cds-icons` v2.1.0
- `@cbhq/ui-mobile-playground` v2.0.0
- `@cbhq/ui-mobile-visreg` v2.0.0

## Breaking Changes

### Prop Names

We renamed some of our component's property names to better align them to accepted [conventions](https://github.cbhq.net/frontend/cds/blob/master/docs/conventions/overview.md) and make our API&apos;s more intuitive and consistent. All migrations are automated with the `cds-migrator`. You can view the full list [here](/resources/deprecations/v6.0.0).

We're also phasing our `dangerouslySet` prefixed prop names and removing that prefix when there is not conflicting styles, eg:

- if a component has `background` and `dangerouslySetBackground` we'll support both properties with `dangerouslySetBackground` having higher priority.
- if a component has `dangerouslySetBackground` and no competing `background` props, we've removed the `dangerouslySet` prefix in CDS 6.

### New Packages

Complex components have been moved into their own standalone packages. Migration was previously optional but is now required in v6. Use the `cds-migrator` to automatically migrate instances and update package dependencies. The migrator will also bump these packages to `v1.0.0`.

- `cds-web-visualization`: Sparkline components
- `cds-mobile-visualization`: Sparkline components

### Removed Packages

We previously isolated some overlay components out of the `@cbhq/cds-web` package into their own `@cbhq/cds-web-overlays` package, however we shifted our strategy and will be reintegrating these components back into the core web package. These migrations were optional in v5, however if you already migrated you can use the cds-migrator to migrate instances back to their original locations. You will need to manually remove `@cbhq/cds-web-overlays` as a dependency for your project after running the script.

For more information on this decision visit [go/web-overlays-strategy](http://go/web-overlays-strategy)

```shell
yarn nx generate @cbhq/cds-migrator:migrate revert-decomp
```

Affected components in `@cbhq/cds-web-overlays`:

- Dropdown, Select, Popover, Tooltip, SidebarMoreMenu, and SelectChip

### Alpha Components Exited

Almost all components and types located in `/alpha` directories have been reintegrated into core packages. All migrations are automated with the `cds-migrator`. See [go/cds-alpha-tdd](http://go/cds-alpha-tdd) for more information on this strategy.

### Stronger Types

We revamped many of the types in our `common`, `web`, and `mobile` packages to use stronger platform-specific type values for event handlers. If you are using any component types from the `common` package, manually migrate to use the type exported from that component's file in either the `web` or `mobile` package.

## Visual Changes

CDS 6 enables frontier styling by default, which introduces visually breaking changes. Anticipated changes are listed below, please review before bashing. Figma with [side by side comparison](https://www.figma.com/file/fm0tnNjeyaJLp0EpbCIlYd/CDS-Frontier-Updates?type=design&node-id=0-1&mode=design&t=84oqxDfNCP7qCni0-0).

:::note
Visual changes include CLS (cumulative layout shift) and changes that could affect accessibility (eg: color contrast).
:::

### Cards

- Removed border
- Background color from `background` -> `secondary`

New cards were introduced in `v5.22.0` which will replace FeatureEntryCard and AnnouncementCard in `v7.0.0`. Migration is optional in CDS 6, but strongly recommended.

:::info
You can use the `cds-migrator` to automatically migrate to the new card suite using `yarn nx generate @cbhq/cds-migrator:migrate gondolin`.
:::
