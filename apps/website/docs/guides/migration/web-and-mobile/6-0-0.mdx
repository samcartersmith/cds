### In this major release we eliminated a lot of tech debt, realigned our component API's to newly accepted conventions, enabled Frontier styling globally, exited alpha components, and improved pressable typing. Almost all breaking changes are automatically migratable with the `cds-migrator` (see [Using the Migrator](#using-the-migrator)).

CDS 6 core packages (`common`, `web`, and `mobile`) are only compatible with the following major package versions (or higher):

- `@cbhq/cds-web-visualization` v1.0.0
- `@cbhq/cds-mobile-visualization` v1.0.0
- `@cbhq/cds-illustrations` v2.1.0
- `@cbhq/cds-icons` v2.1.0
- `@cbhq/ui-mobile-playground` v2.0.0
- `@cbhq/ui-mobile-visreg` v2.0.0

## Breaking Changes

### Prop Names

**View the [full list of changes here](/resources/deprecations/v6.0.0/#props)**

We renamed some component props to make our APIs more intuitive and consistent, and better align them to our [accepted conventions](https://github.cbhq.net/frontend/cds/blob/master/docs/conventions/overview.md). All migrations are automated with the `cds-migrator`.

We're also phasing out prefixing props with `dangerouslySet`. We've removed that prefix when there is not conflicting styles, eg:

- if a component has `background` and `dangerouslySetBackground` we'll support both props with `dangerouslySetBackground` having higher priority.
- if a component has `dangerouslySetBackground` and no `background` props, we've removed the `dangerouslySet` prefix in CDS 6.

### New Packages

The Sparkline components have been moved into their own packages. Migration was previously optional but is now required in v6. Use the `cds-migrator` to automatically migrate instances and update package dependencies. The migrator will also bump these packages to `v1.0.0`.

- `@cbhq/cds-web-visualization`
- `@cbhq/cds-mobile-visualization`

### Removed Packages

The `@cbhq/cds-web-overlays` package has been deprecated in favor of `@cbhq/cds-web`. Migrating to `@cbhq/cds-web-overlays` was optional in v5. You can migrate back to `@cbhq/cds-web` using the cds-migrator. You will need to manually remove `@cbhq/cds-web-overlays` from your package.json after running the migrator.

For more information on the decision to deprecate `@cbhq/cds-web-overlays` please visit [go/web-overlays-strategy](http://go/web-overlays-strategy)

```shell
yarn nx generate @cbhq/cds-migrator:migrate revert-decomp
```

Affected components in `@cbhq/cds-web-overlays` that should now be imported from `@cbhq/cds-web` instead:

- Dropdown, Select, Popover, Tooltip, SidebarMoreMenu, and SelectChip

### Alpha Components Exited

Almost all components and types located in `/alpha` directories have been reintegrated into core packages. All migrations are automated with the `cds-migrator`. See [go/cds-alpha-tdd](http://go/cds-alpha-tdd) for more information on this strategy.

### Stronger Types

We revamped many of the types in our `common`, `web`, and `mobile` packages to use stronger platform-specific type values for event handlers. If you are using any component types from the `common` package, manually migrate to use the type exported from that component's file in either the `web` or `mobile` package.

## Upgraded framer-motion (Web only)

framer-motion has been upgraded from v6 to **v10.18.0** and moved from dependecy to **peerDependency**.

:::info
To mitigate the **peerDependency** change:

- If you were not using Framer Motion in your project, you'll need to install it now.
- If you were using Framer Motion directly in your project, you'll need to update to v10.18.0 as well.

:::

### Breaking changes

There's no breaking changes that impacted CDS.
However, you might encouter breaking changes in your implementation, please see this [upgrade guide](https://www.framer.com/motion/guide-upgrade/) for more info.

### Bundle size reduction

In order to [reduce bundle size](https://www.framer.com/motion/guide-reduce-bundle-size/#available-features),
we replaced the `domMax` feature package with the `domAnimation` package, which resulted in [~17kB](https://buildkite.com/coinbase/frontend-cds/builds/11838#018d4856-3746-4d99-9513-249efbca3eb3) savings in bundle size.
However, this change will remove support for pan/drag gestures and layout animations. If you need those features, you can pass `domMax` into ThemeProvider like so:

```jsx
import { domMax } from 'framer-motion';

<ThemeProvider motionFeatures={domMax}>...your code</ThemeProvider>;
```
