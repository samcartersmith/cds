This page demonstrates the recipes that CDS recommends consumers adopt in order to build out a User Switcher component.

These Recipes are intentionally NOT API-driven and simply serve as guidance on how to create a User Switcher. It is a product team's responsibility to turn these recipes into reusable components.

We have also added demo data to show a more realistic example; note that this is for demo purposes only, we anticipate the consumer will need to modify the content component to work with Relay/GraphQL.

## Anatomy

The User Switcher recipe is composed of the following CDS components:

- [Dropdown](/components/dropdown)
- [ListCell](/components/list-cell)
- [Cell](https://github.cbhq.net/frontend/cds/blob/master/packages/web/cells/Cell.tsx)
- [Collapsible](/components/collapsible)
- [CollapseArrow](/components/collapse-arrow)
- [Avatar](/components/avatar)

Some key details about the abstractions in this recipe:

- Dropdown

  - Dropdown contains the prescriptive layout/overlay logic shared between App and UserSwitcher.

- UserSwitcherContent
  - This is a component that was built for demonstration purposes only.
  - CDS will not expose a component like this, however we expect Identity will implement a product component very similar to this.

```jsx live
function UserSwitcherRecipe({
  title = 'Brian Armstrong',
  description = 'brian.armstrong@coinbase.com',
}) {
  const switcherPositionConfig = {
    placement: 'bottom',
    gap: 1,
  };
  const avatarColorScheme = getAvatarFallbackColor(title);
  return (
    <PortalProvider>
      <FeatureFlagProvider frontierColor frontierButton>
        <HStack spacing={4}>
          <Dropdown
            content={<UserSwitcherContent />}
            width={416}
            maxHeight={674}
            showOverlay
            contentPosition={switcherPositionConfig}
            showMobileModal
          >
            <Pressable backgroundColor="transparent" as="button" borderRadius="standard">
              <HStack gap={1} alignItems="center">
                <Avatar size="xl" alt={title} name={title} colorScheme={avatarColorScheme} />
                <VStack maxWidth={172} minWidth={0}>
                  {description ? (
                    <TextLabel1 overflow="truncate" as="p">
                      {title}
                    </TextLabel1>
                  ) : (
                    <TextHeadline overflow="truncate" as="h3">
                      {title}
                    </TextHeadline>
                  )}
                  {description ? (
                    <TextLabel2 overflow="truncate" as="p">
                      {description}
                    </TextLabel2>
                  ) : null}
                </VStack>
              </HStack>
            </Pressable>
          </Dropdown>
        </HStack>
      </FeatureFlagProvider>
    </PortalProvider>
  );
}
```

## Recipe

Above we encapsulated the content code into a component to cleanly demonstrate the high level architecture of the UserSwitcher Recipe.

The below example is meant to expose the `UserSwitcherContent` Recipe because we have no intention of exposing the `UserSwitcherContent` component.
`UserSwitcherContent` was simply built out for demonstration purposes and we expect it will be different when consumers implement it with GraphQL etc.

```jsx live
function UserSwitcherContentExample() {
  const userSwitcherData = [
    {
      name: 'Brian Armstrong',
      email: 'brian.armstrong@coinbase.com',
      selected: true,
      authenticated: true,
    },
    {
      name: 'Brian Armstrong',
      email: 'barmstrong@bisontrails.com',
      selected: false,
      authenticated: false,
    },
    {
      name: 'Brian Armstrong',
      email: 'bstrong@yahoo.com',
      selected: false,
      authenticated: true,
    },
    {
      name: 'Test Account',
      email: 'brian.armstrong+test@coinbase.com',
      selected: false,
      authenticated: false,
    },
  ];

  // this is the height of 3 account list cells, if there are more, it will scroll vertically
  const userAccountsListMaxHeight = 208;

  const { name, email, avatarUri } = userSwitcherData.find((user) => user.selected);
  const otherAccountsData = userSwitcherData.filter((user) => !user.selected);

  function NoopFn() {
    console.log('pressed');
  }

  function UserAccountListCell({ name, email, selected, authenticated, avatarUri }) {
    const [collapsed, { toggle }] = useToggler(true);

    const userCellOuterSpacingConfig = useMemo(
      () => ({
        spacingHorizontal: 2,
        spacingVertical: 0,
      }),
      [],
    );

    // this will generate a fallback color based on a unique identifier (ideally use UUID, instead of name)
    const avatarColorScheme = getAvatarFallbackColor(name);

    return (
      <VStack background={collapsed ? 'background' : 'backgroundAlternate'} borderRadius="standard">
        <PressableOpacity noScaleOnPress className={insetFocusRing} onPress={toggle}>
          <ListCell
            title={name}
            description={email}
            action={<CollapseArrow degrees={90} collapsed={collapsed} />}
            outerSpacing={userCellOuterSpacingConfig}
            intermediary={!authenticated && <TextLegal as="p">Signed out</TextLegal>}
            priority="middle"
            media={
              <Avatar src={avatarUri} alt={name} name={name} colorScheme={avatarColorScheme} />
            }
            compact
          />
        </PressableOpacity>
        <Collapsible collapsed={collapsed} dangerouslyDisableOverflowHidden>
          <HStack spacingTop={0.5} spacingBottom={1} spacingStart={8} gap={0.5}>
            <Button compact>Sign in</Button>
            <Button compact transparent variant="secondary">
              Remove
            </Button>
          </HStack>
        </Collapsible>
      </VStack>
    );
  }

  return (
    <FeatureFlagProvider frontierColor frontierButton>
      <VStack gap={1} alignItems="center" spacingTop={1} spacingHorizontal={1}>
        {/* selected Avatar should always use the blue colorScheme */}
        <Avatar
          size="xxxl"
          selected
          alt="Brian Armstrong"
          src={avatarUri}
          name="Brian Armstrong"
          colorScheme="blue"
        />
        <VStack alignItems="center">
          <TextTitle3 align="center" as="h3" overflow="wrap">
            {name}
          </TextTitle3>
          <TextBody align="center" as="p" overflow="wrap">
            {email}
          </TextBody>
        </VStack>
        <Button compact transparent onPress={NoopFn}>
          Manage Settings
        </Button>
      </VStack>
      <VStack spacingHorizontal={2}>
        <SectionTitle text="Your other accounts" />
      </VStack>
      <VStack maxHeight={userAccountsListMaxHeight} overflow="auto" gap={0.5} spacingHorizontal={1}>
        {otherAccountsData.map((userProps) => (
          <UserAccountListCell {...userProps} />
        ))}
        <HStack spacingHorizontal={0.5}>
          <Button startIcon="followInactive" transparent>
            Add Account
          </Button>
        </HStack>
      </VStack>
      <Divider spacingBottom={2} />
      <VStack alignItems="center" gap={1} spacingHorizontal={1}>
        <Button variant="secondary" block>
          Sign out
        </Button>
        <Button transparent block>
          Sign out of all accounts
        </Button>
      </VStack>
    </FeatureFlagProvider>
  );
}
```
