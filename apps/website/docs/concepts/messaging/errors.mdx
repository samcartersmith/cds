An error state is a screen that is shown when things go wrong. It is an example of a situation where
the user is getting something other than their desired state. Every error, regardless of cause, becomes
a point of friction for your users and blocks them from moving forward in their experience.

## General guidelines

### Look

- Do not solely use color to signify error. Add an error message or an illustration (sparingly).
- Illustrations are opportunities to communicate and lessen the confusion and frustration, ensuring ease when things aren't working accordingly. Use error state illustrations sparingly but with intent. Consider how many error state illustrations appear on a singular screen. These illustrations set the tone that something is wrong.
- The bigger the component, the more visual noise it creates.
- Use the color red sparingly.
- Search for these illustrations in our search with the name of the error (e.g. "404", "500", etc.) or "error" in the search bar. We have an ‘error’ icon which is in its name.

### Sound

- Error messages should be clear, concise, and use consistent vocabulary.
- Error messages should be written according to local standards and guidelines.
- Use sentence case capitalization.
- For country-specific requirements, reach out to #ask-localization on Slack.

[Coinbase Content Design Guide](https://docs.google.com/document/d/1QnxOOjhqUb8ENrGt3hQTTDCDxnCPRyEfkO-U-ftm4uc/edit#heading=h.xo750yt3a4gs) <br />

### Feel

- Guide users through areas where they need to focus.
- Consistent use of animation to set clear expectations.
- Always consider the user's emotional state, as seeing these symbols/icons are like visual alerts as they set the tone for the incident at hand.
- Only use dramatic animation for urgent messages.
- Use hint motion sparingly but with intent. Hint motion should be applied to components or surfaces that need immediate attention. Adding or combining various hints system in a single view can control the severity of the tone. Overstacking can causing a negative experience.

## Severity of error

Severity is measured by friction, the degree the error blocks the user from moving on, and level of ease to remedy.

As there are a spectrum of error states, the choice of illustration is incredibly important. Illustrations are used for medium to high error messages.
