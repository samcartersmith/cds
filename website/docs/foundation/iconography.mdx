We use iconography as both an expression of the brand and a navigation tool. We define successful iconography as symbols that are recognized immediately, internationally, and without prior study.

## Using Iconography

Choosing whether to use an icon, an illustration or text can be complex. An icon typically aims to improve usability through one or more of the following goals:

- Facilitate navigation by using universal icons in standard product paradigms (e.g. tab bars or navigation).
- Reduce cognitive load by condensing less relevant information or repeating content.
- Improve legibility by breaking up content into smaller, easier to understand pieces.
- Reach a wider audience by visually reinterpreting text for non-English speakers.
- Focus attention by creating a visual hierarchy.
- Function triumphs aesthetics, we aim for clarity and recognition when using icons as a communication tool.

## Icon Treatments

### Do’s and Don’ts

Misused iconography can reduce usability. Try to avoid the following when using iconography:

Icons as an ornamental element / too many icons on one screen, try without icons or limit the key interactions.

<DoDont>
<DoExample img={{light:"/img/foundation/iconography/ornamental_do.svg", dark:"/img/foundation/iconography/ornamental_do_dark.svg"}} >
  By removing the icons in the form fields, it allows the support copy to shine, and give focus to the right parts of the experience.

</DoExample>
<DontExample img={{light:"/img/foundation/iconography/ornamental_dont.svg", dark:"/img/foundation/iconography/ornamental_dont_dark.svg"}}>
  Icons in form fields can be decorational if not interactable, additionally the icons can add more unnecessary visual weight.
</DontExample>

</DoDont>

Icons and illustrations should not be together on a single screen, choose either the illustration or the icon.

<DoDont>
<DoExample img={{light:"/img/foundation/iconography/illustration_do.svg", dark:"/img/foundation/iconography/illustration_do_dark.svg" }}>
Choosing either illustration or iconography to communicate the feature offers more focus.

</DoExample>
<DontExample img={{light:"/img/foundation/iconography/illustration_dont.svg", dark:"/img/foundation/iconography/illustration_dont_dark.svg" }}>
By having both an illustration and iconography can be distracting.

</DontExample>

</DoDont>

## Labels

We recommend labels to be paired with iconography as much as possible. As much as the icon working alone seems aesthetically pleasing, it can confuse and frustrate users, adding to less accessible product and experience.

<DoDont>
  <DoExample
    img={{
      light: '/img/foundation/iconography/labels_do.svg',
      dark: '/img/foundation/iconography/labels_do_dark.svg',
    }}
  >
    Adding labels to icons offers a sense of clarity and accessibility for users.
  </DoExample>
  <DontExample
    img={{
      light: '/img/foundation/iconography/labels_dont.svg',
      dark: '/img/foundation/iconography/labels_dont_dark.svg',
    }}
  >
    Icons without labels struggle with glanceability and visual comprehension.
  </DontExample>
</DoDont>

Labels can be removed when paired with icons; we recommend this for only universal icons. Universal icons are rare since they need to have a shared understanding across many cultures, ages, languages, or locations.

<Image
  light="/img/foundation/iconography/labels_universal.svg"
  dark="/img/foundation/iconography/labels_universal_dark.svg"
/>

Examples of universal icons are the favorite (star), like (heart), hamburger (three lines) and edit (pencil) icon. As these icons are a rarity, we still recommend using a label whenever possible.

## Accessibility

When pairing icons with UI, it's essential to keep the color contrast ratio to at least 3:1. Especially as you scale and add color to iconography, it's crucial to keep this ratio in mind.

<DoDont>
<DoExample img={{light:"/img/foundation/iconography/coloraccessibility_do.svg", dark:"/img/foundation/iconography/coloraccessibility_do_dark.svg" }}>
The use of gray here passes color contrast at 5.13:1.

</DoExample>
<DontExample img={{light:"/img/foundation/iconography/coloraccessibility_dont.svg", dark:"/img/foundation/iconography/coloraccessibility_dont_dark.svg" }}>

</DontExample>
</DoDont>

## Bordered / Container Icons

Icons in borders should be used sparingly, as it optically shrinks the icon inside of the shape, and when overused it can be overly decorative. In most cases we recommend just the icon. Although, effective use of icons and borders is our contacts list.

## Icons in a List

Adding icons to a list helps with parsing information in a glanceable and scannable way. Users should be consuming this information in bite-size amounts, as the icon never works alone in a list format.

### Size and Weight

We've designed our icons to match the headline to carry the same visual weight when reading down the list; we recommend our default size of 24px. The icon can be a different size; it's essential to refer to the list item's headline's visual weight.

<DoDont>
<DoExample img={{light:"/img/foundation/iconography/list_do.svg", dark:"/img/foundation/iconography/list_do_dark.svg" }}>
Use consistent visual weight.

</DoExample>
<DontExample img={{light:"/img/foundation/iconography/list_dont.svg", dark:"/img/foundation/iconography/list_dont_dark.svg" }}>
Use inconsistent visual weight.

</DontExample>
</DoDont>

### Alignment

The icon should be top-aligned with the headline to serve the readability best. If the icon is in the middle of the headline and body text, it will create tension when reading down the list.

<DoDont>
<DoExample img={{light:"/img/foundation/iconography/alignment_do.svg", dark:"/img/foundation/iconography/alignment_do_dark.svg" }}>
Easy read from icon to headline and rest of the copy.

</DoExample>
<DontExample img={{light:"/img/foundation/iconography/alignment_dont.svg", dark:"/img/foundation/iconography/alignment_dont_dark.svg" }}>
Misalignment when reading the icon with the rest of the copy.

</DontExample>
</DoDont>

### The Number of Icons

When referring to a list of icons, users can only contain between 3-5 items of information.

## Information or Question Mark Icon

When thinking about sharing helpful information with users, there are two icons that we can use: the information or question mark icon. However, both carry two distinct use cases.

<Image
  light="/img/foundation/iconography/info_questionmark.svg"
  dark="/img/foundation/iconography/info_questionmark_dark.svg"
/>

### Information

This icon is most commonly paired with our tooltip component to share a snippet of additional information without taking the user away from the current screen.

### Question Mark

When users need more in-depth information (i.e. documentation, FAQs, support pages, etc.), we should use the question mark icon. This experience entails that users are taken to another screen to seek more detailed information.

## Using icons in dense interfaces

Icons are sized, scaled and designed for these particular contexts to help communicate with the limited space provided. With this in mind, we recommend using universal icons in a concise manner in these contexts, as there isn't room for a label to accompany the icon.

<DoDont>
<DoExample img={{light:"/img/foundation/iconography/dense_do.svg", dark:"/img/foundation/iconography/dense_do_dark.svg" }}>
Keep the icon universal and simple to allow the icon to thrive in the dense information area.

</DoExample>
<DontExample img={{light:"/img/foundation/iconography/dense_dont.svg", dark:"/img/foundation/iconography/dense_dont_dark.svg" }}>
Detailed or unfamiliar icons are not visual comprehensible when used in a dense space.

</DontExample>
</DoDont>

## Contributing OR Requesting an Icon

You can make an icon request here: go/iconrequest. Please make sure you’re requesting support at least 2 weeks prior to the due date.

We’re still writing up the documentation around contributing / creating an icon into CDS and that will be coming soon.

### Approvals and Finalization

We will work with you on the best option for the icon request. However, the Illustration Team will make the final call on quality to ensure consistency with the system. Once this is done, the Illustration Team can add the icon into CDS.
