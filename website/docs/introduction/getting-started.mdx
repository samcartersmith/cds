## Using CDS inside the [`mono/repo`](https://github.cbhq.net/mono/repo)

### For web projects

1. Set up a project level `.babelrc.js` and include `linaria/babel` at the end of the presets list. CDS relies on [linaria](https://github.com/callstack/linaria) for styling on the web.

```js
module.exports = {
  presets: ['linaria/babel'],
  plugins: ['@babel/plugin-proposal-class-properties'],
};
```

2. In the bazel rule for building your application, make sure to include `"//eng/shared/design-system:web_source"` as part of the `data` argument.

## Using CDS outside the [`mono/repo`](https://github.cbhq.net/mono/repo)

1. Add `@cb:registry=https://registry-npm.cbhq.net` to your `.npmrc`.
2. Install the CDS packages you need

```bash
yarn add @cds/theme # must have
yarn add @cds/web # if using web components
yarn add @cds/mobile # if using mobile components
yarn add @cds/icons # if using CDS icons
```

## Building with CDS

1. Add `ThemeProvider` to the root of your React application. Choose a scale for your application from the seven scales supported by CDS.

```tsx
import { ThemeProvider } from '@cds/theme'

export const App = () => {
  <ThemeProvider scale="large">
    <ErrorBoundary>
      <AppRoutes />
    </ErrorBoundary>
  <ThemeProvider>
}
```

If you have components that are created outside of the app's main DOM tree, such as `Modal`s that use `createPortal`, you will need to add `ThemeProvider` to the root of the portal element as well.

2. Import components from `@cds/web` or `@cds/mobile`.
3. To figure out what CDS primitive or component is being used in the design, you can inspect the target in Figma and look for component label, color palette name, or spacing size.
