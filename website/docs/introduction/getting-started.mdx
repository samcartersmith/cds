## Using CDS inside the [`mono/repo`](https://github.cbhq.net/mono/repo)

### For web projects

1. In the bazel rule for building your application, make sure to include `"//eng/shared/design-system:web_source"` as part of the `data` argument.

## Using CDS outside the [`mono/repo`](https://github.cbhq.net/mono/repo)

1. Add `@cbhq:registry=https://registry-npm.cbhq.net` to your `.npmrc`.
2. Install the CDS packages you need

```bash
yarn add @cbhq/cds-web # if using web components
yarn add @cbhq/cds-mobile # if using mobile components
```

## Building with CDS

1. Import `globalStyles`. This includes the font face declarations,

```tsx
import '@cbhq/cds-web/globalStyles';
```

2. Add `ThemeProvider` to the root of your React application. Choose a scale for your application from the seven scales supported by CDS.

```tsx
import { ThemeProvider } from '@cbhq/cds-web/system/ThemeProvider';

export const App = () => {
  <ThemeProvider scale="large">
    <ErrorBoundary>
      <AppRoutes />
    </ErrorBoundary>
  <ThemeProvider>
}
```

If you have components that are created outside of the app's main DOM tree, such as `Modal`s that use `createPortal`, you will need to add `ThemeProvider` to the root of the portal element as well.

3. Import components from `@cbhq/cds-web` or `@cbhq/cds-mobile`.
4. To figure out what CDS primitive or component is being used in the design, you can inspect the target in Figma and look for component label, color palette name, or spacing size.

   - example 2. Use `postive` palette color for the background. 🌙 indicates that the palette is for dark mode. In most cases, light and dark mode can have the same styling.
