## New Shape: pill buttons

<Image light="/img/frontier/pillbuttons.png" dark="/img/frontier/pillbuttons_dark.png" />

One of the visual cornerstones for Frontier is pill buttons. Now with the `frontierButton` [feature flag](/components/feature-flag-provider), you’ll receive the new pill buttons on all CDS Buttons automatically!

In Design, we don’t have Feature Flags, and so you’ll be able to access these via the new component “Button Frontier”! We’ll sunset the non-frontier Button styles at some point in 2022, and we’ll shift fully to Frontier button styles. \*The specific sunset date will be identified and announced with ample time for product teams to prepare.

## Want to start using Frontier buttons now?

### Engineers

Leverage the new [CDS FeatureFlagProvider](/components/feature-flag-provider) to toggle on the new button styles. CDS feature flags aren't backend driven or anything fancy so they will need to be configured per project/platform. This is similar to how a ThemeProvider can be customized per product and per platform.

The CDS FeatureFlagProvider is a simple React Context that is used internally in CDS components to conditionally render certain styles or changes.

The goal of the FeatureFlagProvider is to provide a mechanism for teams to opt in/out of potentially disruptive changes. The `frontierButton` feature flag is the flag you will want to turn on to access the new button styles.

We recommend working with your Product team to coordinate how you want to roll these changes out. Since the FeatureFlagProvider is a simple on/off mechanism the CDS feature flags can be driven by an experiment, product specific feature flag or kill switch.

To demonstrate how this works we’ve set up a CDS Frontier page in the [Retail Mobile Alpha Debug menu](http://go/beta). You can access it by going to Settings > Debug > CDS Frontier and toggling on the feature flags you would like to test out and then navigating around the app to see how the new styles propagate.

Please note, this CDS Frontier Debug Menu is just for demonstration purposes and these settings will not propagate across accounts or mobile/web. This Debug Menu simply saves your feature flag preferences to local storage and updates the [CDS FeatureFlagProvider](/components/feature-flag-provider). To carry these changes more permanently the CDS feature flags would need to be turned on permanently or powered by the backend (via experiment, etc).

### Designers

We’ve added [Frontier Button into Figma](https://www.figma.com/file/SWoyy3B5IkEpMvk60Lb4V6/CDS-Normal-%F0%9F%8C%9E?node-id=561%3A2). We want to prevent any confusion, so your mocks will not auto update to the new Frontier Button until you’re ready to do so. When ready, just swap instances or drag the new Frontier Button into your mock.

We’ve also made some improvements to the Figma interface. This will expose more functionality that was available in code but not available in Figma, so you’ll see some new toggles but know that those options are also available outside of Frontier.

In Q1, we’ll ask everyone to migrate all button instances to Frontier buttons in Figma. This is Figma only, so your engineer will not need to mirror this in code. We’ll rename Frontier Button to Button and rename the current to Button (Deprecated) and no longer support bugs.
