## Intro

Frontier is an upcoming update to CDS that’s designed to support the implementation of the Unified App North Star visual design language. CDS will update specific primitives and components. Frontier is the first update to CDS, and we’ll be piloting new programs and techniques that will aim to be less disruptive to product teams.

### Updates

As updates are released, we’ll link to them [here](#milestones).

All existing and new CDS components will be Frontier-ready, so they’ll work automatically with the new capabilities that Frontier will offer.

## Rollout

There will be two types of rollouts in Frontier.

### Additive

With Frontier, we’ll be adding new capabilities into CDS. With these updates, we’ll be adding them to CDS immediately, and they will not be behind a feature flag.

### Modification

There also will be aspects of Frontier that change existing CDS resources. For Q4, those changes will be entirely opt-in. Teams will be able to test and roll out these changes as they see fit.

To help with this, we’ll be releasing six feature flags:

- Frontier typography
- Frontier buttons
- Frontier color
- Frontier card
- Frontier sparkline

See how to test and enable [Frontier feature flags](/components/feature-flag-provider).

To make testing even easier, we've added a CDS Frontier menu item to the debug menu in CB Alpha. To get CB Alpha, follow these [instructions](http://go/beta).

### Figma resources

We’ll be adding all Additive Frontier changes to Figma. Modification Frontier changes will be available in branches of CDS Libraries and will be merged according to on team needs.
