> [@cbhq/cds-(web|mobile)/color/useAccessibleForeground](https://github.cbhq.net/mono/repo/blob/master/eng/shared/design-system/web/color/useAccessibleForeground.ts)

Does the following checks on a given color:

1. Check if color meets contrast criteria.
2. If pass, return color passed in.
3. If fail, check saturation.
4. If gray color return `gray100`.
5. If non gray color return closest CDS spectrum alias (i.e. `orange60`).

```jsx live
function Example() {
  const foreground = useAccessibleForeground({ color: 'yellow', usage: 'normalText' });
  return (
    <ExampleWithThemeToggles>
      <p style={{ color: foreground }}>
        Yellow text converted to accessible color based on current background color
      </p>
    </ExampleWithThemeToggles>
  );
}
```

```jsx live
function Example() {
  const foreground = useAccessibleForeground({ color: 'yellow', usage: 'largeText' });
  return (
    <ExampleWithThemeToggles>
      <h1 style={{ color: foreground }}>
        Yellow text converted to accessible color based on current background color
      </h1>
    </ExampleWithThemeToggles>
  );
}
```

## Web considerations

For CDS web you cannot pass in the palette css variable from the palette object (i.e. palette.primary).

Instead you will need to convert the palette value to an rgba string. This is because the palette object for web is a static object which maps a palette alias to its CSS variable.

On mobile, CSS variables are not supported so the object returned from `usePalette` is an object of rgba strings so this is not an issue.

To get the rgba string of a palette alias you'll need to use:

1. usePaletteConfig - get the palette config value.
2. usePaletteValueToRgbaString or - convert the palette config value to rgba string.

### First argument is a valid color (hex, rgb, rgba).

Some examples:

- `rgb(255, 255, 255)`
- `rgb(10%, 20%, 30%)`
- `rgba(255, 255, 255, 0.4)`
- `rgba(10%, 20%, 30%, 0.4)`
- `hsl(120, 50%, 20%)`
- `hsla(120, 50%, 20%, 0.4)`
- `#ffeeaa`
- `#fea`
- `#ffeeaa22`
- `#fea2`
- `steelblue`

### Second argument is usage of color

- `normalText`
- `largeText`
- `graphic`
