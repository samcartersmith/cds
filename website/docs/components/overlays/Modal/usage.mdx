### Get Started
1. Add `PortalProvider` to the root of your app. Import it from `import { PortalProvider } from '@cbhq/cds-web/overlays/PortalProvider';`;
2. Recommended to place your `Modal` inside the same parent as the trigger, compose with `ModalHeader`, `ModalBody`, and `ModalFooter` as you need.
3. Wire up `visible` and `onRequestClose` to control visibility.

### Default Modal
- Modal is using React Portal by default, which will port the node to the end of the `body` tag. You can add `disabledPortal` to disable this behavior.

- Although `ModalHeader`, `ModalBody`, `ModalFooter` are optional sub-components, it's **recommended** to compose them like below for consistency:
```
<Modal>
	<ModalHeader />
	<ModalBody>
		<AnyContentYouWant />
	</ModalBody>
	<ModalFooter />
</Modal>
```

```jsx live
// import { PortalProvider } from '@cbhq/cds-web/overlays/PortalProvider';

function DefaultModalExample() {
  const [visible, { toggleOn, toggleOff }] = useToggler();
  const [scrolling, { toggle: toggleScrolling }] = useToggler();

  return (
    <ExampleWithThemeToggles>
      <PortalProvider>
        <ButtonGroup>
          <Button onPress={toggleOn}>Basic Modal</Button>
          <Switch onChange={toggleScrolling} checked={scrolling}>
            Show long content
          </Switch>
        </ButtonGroup>
        <Modal
          visible={visible}
          onRequestClose={toggleOff}
        >
          <ModalHeader title="Basic Modal" /> {/* close action is inherited from onRequestClose */}
          <ModalBody>
            <TextBody as='p'>
              {loremIpsum.repeat(scrolling ? 50 : 1)}
            </TextBody>
          </ModalBody>
          <ModalFooter
            primaryAction={<Button onPress={toggleOff}>Save</Button>}
            secondaryAction={<Button variant="secondary"  onPress={toggleOff}>Cancel</Button>}
          />
        </Modal>
      </PortalProvider>
    </ExampleWithThemeToggles>
  );
}
```

### Programmatically Control Modal

`PortalProvider` is a convenient way for adding anything to the CDS portal. The CDS portal is what CDS uses to centralize all elements that need a portal and handles the zIndex of those pieces.
You can control Modal visibility programmatically using `openModal()` and `closeModal()` from the `useModal()` hook which consumes `PortalProvider` under the hood.

**NOTE:** PortalProvider is unstable on mobile and will break if multiple portals implements React Native Modal. This is a known issue with React Native Modal which doesn't support multiple modals display at the same time.

```jsx live
// import { PortalProvider } from '@cbhq/cds-web/overlays/PortalProvider';
// import { useModal } from '@cbhq/cds-web/overlays/useModal';

function PortalModalExample() {
  function ModalExample() {
    const { openModal } = useModal();

    const handlePress = useCallback(
      () => openModal(
        <Modal visible onRequestClose={() => { console.log('close modal') }}>
          <ModalHeader title="Basic Modal" />
          <ModalBody>
            <TextBody as="p">{loremIpsum}</TextBody>
          </ModalBody>
          <ModalFooter
            primaryAction={<Button>Save</Button>}
            secondaryAction={<Button variant="secondary">Cancel</Button>}
          />
        </Modal>
      ),
      [openModal],
    );

    return (
      <ExampleWithThemeToggles>
        <Button onPress={handlePress}>Open Modal</Button>
      </ExampleWithThemeToggles>
    );
  }

  return (
    <PortalProvider>
      <ModalExample />
    </PortalProvider>
  );
}
```
