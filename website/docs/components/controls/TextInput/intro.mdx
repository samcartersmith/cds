Text Inputs enable users to type, enter, and/or edit text. Text Field is customizable and has the ability to support various types of objects at the start and end of the input. This component is most commonly used in forms.

## Anatomy of Inputs

<span>&nbsp;</span>

1. **Label:** Title / brief description of input
2. **Container:** Container of input
3. **Input Text:** Content will vary based on type of input
4. **Helper Text** (Optional): Assistive message about the input

## Guidance

- Use Text Inputs for free form responses or when we can’t predict a user’s answer.
- Use another component like Dropdown, Date Input, or one of our Group Selection Control components if a user is selecting from a set of options.

## Content Guidelines

- Always include a label.
- Write Input Labels, Placeholder Text, and Helper Text in sentence case.
- Use Helper Text to show hints, formatting, and requirements.
- Use Helper Text instead of Placeholder Text.
- Do not explain what is wrong but rather offer a solution. Error messages should be clear and concise.
- For more info on writing text inputs, check our [Content Design Guide](https://docs.google.com/document/d/1QnxOOjhqUb8ENrGt3hQTTDCDxnCPRyEfkO-U-ftm4uc/edit#heading=h.m7ffxiih1sf1).

## Sizing & Scale

Text Inputs are available in Dense and Normal scales. We also offer a compact option for smaller spaces and dense experiences.

When stacking inputs, use spacing 3 (24px). For dense experiences, use spacing 2 (16px).

<Box justifyContent="center">
  <Image
    light="/img/components/inputs/TextInput/sizing_and_scale_breakdown.png"

Heights are predetermined and based on scale. Please avoid customizing the height of the input unless necessary. The height value has been selected to align with our Button components.

```jsx live
<HStack gap={2}>
  <TextInput
    label="Email"
    placeholder="satoshi@nakamoto.com"
    helperText="Please enter a valid email address"
  />
  <Box spacingTop={4}>
    <Button variant="primary">Submit</Button>
  </Box>
</HStack>
```

Input widths can be customized but we recommend being consistent. This is especially important when grouping Text Inputs in a Form.

```jsx live
function FormExample() {
  const gap = 3;

  const onSubmit = useCallback(e => {
    e.preventDefault();
    console.log(e.currentTarget.nodeValue);
    // eslint-disable-next-line no-alert
    alert('Submitted');
  }, []);

  return (
    <form onSubmit={onSubmit} action={undefined}>
      <VStack gap={gap}>
        <TextInput
          label="Street address"
          placeholder="4321 Jade Palace"
          helperText="Please enter your primary address."
        />
        <TextInput label="Unit #" aria-required="true" />
        <HStack gap={gap}>
          <TextInput label="City/town" width={70} />
          <TextInput label="State" width={30} />
        </HStack>
        <HStack gap={gap}>
          <TextInput label="Postal code" width={40} />
          <TextInput label="Country" width={60} />
        </HStack>
        <ButtonGroup>
          <Button type="submit">Save</Button>
        </ButtonGroup>
      </VStack>
    </form>
  );
}
```
