# Coinbase Design System

Welcome to CDS!

If you are NOT a team member on CDS you are probably looking for the CDS website ([go/cds](https://cds.cbhq.net/)).

If you ARE on CDS you will find this README very helpful. 
Further, if you are new to CDS, completing the checklist tasks will be a great way to onboard.

1. [CDS Confluence Home](https://confluence.coinbase-corp.com/pages/viewpage.action?spaceKey=CDS&title=CDS+Home) - If you aren't finding what you need in our github documentation this is a great place to look.
>- Click into the different links and try to get a feel for their purpose.


2. [The CDS Website](https://cds.cbhq.net/) - This is where we document CDS best practices, principles, components and more for other teams to consume. It is our primary communication channel to CDS consumers.
>- It containes a lot of information so try to get a feel for the organization of the website. Don't get too overwhelmed with all of the information up front.

3. [Contributing to the CDS Codebase](./CONTRIBUTING.md) - This document is great for setting up your CDS dev environment and learning more about what your dev cycles will look like.

>- Run our mobile playground locally and understand its purpose
>- Run storybook locally and understand its purpose
>- Run the CDS website locally and understanding its purpose
>- Uncomment [this code](https://github.cbhq.net/mono/repo/blob/f09787e22aab7da2ab6d2e9d995901337d06bf53/eng/shared/design-system/website/sidebars.js#L117) to read about our teams core development principles and coding conventions.

4. [Component Lifecycle Doc](https://docs.google.com/document/d/1L9Zk2D7oKuJxq__FVg0GpMFwIZYhcJJA5fSVlkwe-X0/edit#) - You will likely be building new components in CDS. This document gives a good overview of process associated with building new components in CDS
>- Understand why this process is so important to your job
>- Confirm you understand your responsibilities as an engineer (or designer)



5. [On-Call](https://confluence.coinbase-corp.com/pages/viewpage.action?spaceKey=CDS&title=CDS+Oncall+Runbook) - We rotate the on-call DRI on a weekly cadence.
- Read the responsibilities section of the confluence page to get a feel for some of the processes our team must maintain.
- Have an idea of what needs to be done for release and icon updates.
