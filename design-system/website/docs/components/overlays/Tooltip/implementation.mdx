### disablePortal <Badge variant="web" /> <Badge variant="danger" order={2} />

By setting this to true you are essentially opting out of CDS zIndex management. By default the subject is rendered in a portal. If set to true the subject will be rendered in the DOM hierarchy of the parent component.

<ApiDetails
  componentName="Tooltip"
  propertyName="disablePortal"
  required={false}
  defaultValue="false"
  mobileOptions={[]}
  webOptions={['false', 'true']}
/>

<ComponentDetails componentName="Tooltip" templateName="prop-disablePortal" />

### gap

This value corresponds to how big the gap between the subject and the tooltip is. We do not encourage usage of this prop. But it is enabled for special cases as an escape hatch.

<ApiDetails
  componentName="Tooltip"
  propertyName="gap"
  required={false}
  defaultValue="1"
  mobileOptions={['0', '1', '2']}
  webOptions={['0', '1', '2']}
/>

<ComponentDetails componentName="Tooltip" templateName="prop-gap" />

### yShiftByStatusBarHeight <Badge variant="mobile" /> <Badge variant="danger" order={2} />

This prop acts as the escape hatch to mitigate this; by setting true one can shift the y position of the tooltip to account for this bug.
This prop only takes affect on Android and should only be used for a very specific measurement edge case described here. There is a known RN Modal bug where screen measurements are offset by the status bar when statusBarTranslucent=false.

<ApiDetails
  componentName="Tooltip"
  propertyName="yShiftByStatusBarHeight"
  required={false}
  defaultValue="false"
  mobileOptions={['false', 'true']}
  webOptions={[]}
/>

<ComponentDetails componentName="Tooltip" templateName="prop-yShiftByStatusBarHeight" />
