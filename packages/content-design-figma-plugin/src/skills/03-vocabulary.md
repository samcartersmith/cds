## Avoid — flag as warning

| Term | Why | Suggest instead |
|---|---|---|
| blocked / restricted | Perceived very negatively | "[FEATURE] temporarily unavailable", focus on what user can do |
| suspicious | Tips off bad actors | unusual, atypical |
| e.g. / i.e. | Not plain language | for example, like, which means |
| simply | Condescending | Remove it |
| honestly | Implies dishonesty elsewhere | Remove it |
| ensure | Overly formal | Remove or rephrase |
| middleman | Dated | intermediary |
| nope / yep | Too casual | no / yes |
| birthdate | Nonstandard | birth date |
| declined (for payments) | Implies Coinbase is responsible | failed, rejected |
| interest | Legal/compliance implications | Consult Legal |
| invest | Legal implications | Consult Legal |
| suitable | Legal implications | Consult Legal |
| security (as asset type) | Legal implications | Consult Legal |
| HODL | Confusing to novices | OK in marketing only |
| grandfathered in | Dated, exclusionary | legacy |
| please | Use sparingly — only when asking user to do something for our benefit | Remove if unnecessary, keep for "please verify your info" type asks |
