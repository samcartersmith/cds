# Product Module: Account Restrictions (Code Yellow)

NOTE: The action-oriented and "nudging action" patterns in this module apply ONLY to account restriction screens where the user CAN take action. They do NOT apply to error headlines that communicate a system failure or outcome (e.g., "We couldn't verify your Social Security number"). Those headlines are protected and must not be reframed. See the Protected error headlines rule in the errors module.

> Apply these rules when reviewing restriction/blocked/locked messaging screens.

## Principles

Positivity, clarity, proactivity. Every restriction message should reflect these.

## Pattern for full-screen takeovers

- **Title:** Focus on action if possible. If no action, explain the state.
- **Body:** What happened → account status → what user CAN still do → how to fix (if fixable)
- **CTA:** Action verb
- **CTA2:** Alt action or dismiss

## Four message types

### Action required, high risk

- Tone: reassuring, nudging action
- Title: "Do this to continue using Coinbase" or "We did this to protect you"
- Body: "You need to [action] to continue. You still have access to x, y, z."

### Action required, low risk

- Tone: informational, nudging action
- Title: "Do this (before [date])"
- Body: "You need to [action] by [date]. You can still x, y, z."

### No action, temporary

- Tone: reassuring, informational
- Title: "We did this to protect you"
- Body: "This is temporary. Check your email."

### No action, permanent

- Tone: reassuring, informational
- Title: "We did this to protect you"
- Body: "You can still x, y. Check your email."

## Severity: error — banned words in restriction context

- "blocked" → "[FEATURE] temporarily unavailable"
- "restricted" / "restriction" → rephrase around what users CAN do
- "suspicious" → "unusual" or "atypical"
- "ensure" → remove
- "For security reasons" → explain the specific reason

## The Everything Exchange

- Main CTA: "Trade" when specific actions aren't available
- Don't say "apply for X" — say "Get started with…"
